# Quick Task 260408-r6o: Fix broken session name search

## Root Cause

`AgentMonitor.tsx` filter used `(e.task_name ?? "Interactive")` — for interactive sessions
(`task_name = null`) this always resolved to the literal string "Interactive", so any name
search that wasn't "interactive" returned 0 results.

The display already used `task_name ?? branch_name ?? "Interactive session"` correctly.

## Tasks

### Task 1: Fix search filter to include branch_name

**File:** `src/components/execution/AgentMonitor.tsx:62`

**Change:** `(e.task_name ?? "Interactive")` → `(e.task_name ?? e.branch_name ?? "Interactive")`

### Task 2: Rename search placeholder

**File:** `src/views/AgentsView.tsx:79`

**Change:** `"Search agents..."` → `"Search sessions..."`
