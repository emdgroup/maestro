In `src-tauri/src/git/mod.rs`, replace the `list_branches_local` stub (lines 141-147) with a real implementation that runs `git branch -a` via `std::process::Command` in the given path directory, parses the output to extract branch names (strip leading whitespace, asterisk for current branch, `remotes/origin/` prefix for remote branches), and returns `Vec<String>`. Also add a helper `fn get_current_branch_local(path: &str) -> Result<String, String>` that runs `git rev-parse --abbrev-ref HEAD` to get the current branch name.

2. In `src-tauri/src/ipc/task_handlers.rs`, add a new IPC command:
   ```rust
   #[tauri::command]
   #[specta::specta]
   pub async fn list_project_branches(
       app_state: State<'_, Arc<AppState>>,
       project_id: i32,
   ) -> Result<(Vec<String>, String), String>
   ```
   This command:
   - Looks up the project by ID to get its path and connection_id
   - Builds a `GitConnection` (using `get_git_connection` from `db` module if available, or inline logic)
   - For local projects: calls `git::list_branches` and a new `git::get_current_branch` that dispatches local vs remote
   - Returns a tuple of `(branches, current_branch)`
   - If git fails (e.g. not a git repo), return `(vec![], "main".to_string())` as fallback
