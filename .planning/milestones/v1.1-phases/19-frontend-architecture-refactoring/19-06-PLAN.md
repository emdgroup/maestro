```bash

  grep -v "from ['\"]react" | \
  grep -v "from ['\"]zustand" | \
  grep -v "from ['\"]@tauri-apps" | \
  grep -v "from ['\"]classnames" | \
  grep -v "from ['\"]sonner" | \
  grep -v "from ['\"]react-hook-form" | \
  grep -v "from ['\"]class-variance-authority" | \
  grep -v "from ['\"].*/ui/" | \
  grep -v "from ['\"]\\./use" > /tmp/imports_to_fix.txt
```

2. Review /tmp/imports_to_fix.txt for relative imports that should be @/ paths

3. Fix any remaining patterns:
   - `from "../components/X"` → `from "@/components/X"`
   - `from "../../components/X"` → `from "@/components/X"`
   - `from "./X"` within same domain is OK (stay local for internal refs)
   - `from "../utils/X"` → `from "@/utils/X"`
   - `from "../store/X"` → `from "@/store/X"`

4. Strategy for local internal imports:
   - ALLOW: `from "./TaskCard"` (same folder)
   - ALLOW: `import "./index"` (self import)
   - CONVERT: `from "../../kanban/TaskCard"` → `from "@/components/kanban"`
   - CONVERT: `from "../components/X"` → `from "@/components/X"`

5. Use VSCode Find & Replace (regex mode) to do bulk replacements:
   - Pattern: `from ['\"]\\.\\./(hooks|lib|utils|components|store|types|services|views)` → `from "@/$1"`
   - Pattern: `from ['\"]\\.\\./(components|store|types|services|views|utils)` → `from "@/$1"`

6. After replacements, run TypeScript again: `pnpm tsc --noEmit`

7. If errors remain, fix them manually

```bash
  grep -v "from ['\"]react" | \
  grep -v "from ['\"]zustand" | \
  grep -v "from ['\"]@tauri-apps" | \
  grep -v "from ['\"]" src/components/ui | \
  grep -v "^.*:\\s*from ['\"]\\./[^.]*['\"]" | \

  wc -l
```
This should return 0 (or very small number for allowed patterns).
