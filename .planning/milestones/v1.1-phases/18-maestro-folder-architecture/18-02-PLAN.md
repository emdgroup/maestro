  <files>src-tauri/src/db/project_storage.rs</files>
  <action>
    Create new file `src-tauri/src/db/project_storage.rs` with the following structure:

    1. Import statements:
       ```rust
       use std::path::Path;
       use crate::models::{ProjectConfig, ProjectState, TaskSnapshot, WorktreeSnapshot, Task, Worktree};
       ```

    2. Function: `create_project_maestro_folder(project_path: &str) -> Result<(), String>`
       - Purpose: Initialize .maestro directory structure for new projects
       - Logic:
         * Use Path::new(project_path).join(".maestro")
         * Call std::fs::create_dir_all() to create directory
         * Return Ok(()) on success, Err(msg) on failure
       - Error message should include project_path and OS error details

    3. Function: `export_config_to_settings(config: &ProjectConfig, project_path: &str) -> Result<(), String>`
       - Purpose: Save project configuration to .maestro/settings.json
       - Logic:
         * Call config.save_to_project(project_path)
         * Return result unchanged
       - This is essentially a wrapper for ProjectConfig::save_to_project for clarity

    4. Function: `export_state_to_file(state: &ProjectState, project_path: &str) -> Result<(), String>`
       - Purpose: Save project state to .maestro/state.json
       - Logic:
         * Call state.save_to_project(project_path)
         * Return result unchanged

    5. Function: `load_project_config(project_path: &str) -> Result<ProjectConfig, String>`
       - Purpose: Load configuration from .maestro/settings.json (or provide defaults if not found)
       - Logic:
         * Try ProjectConfig::load_from_project(project_path)
         * If error (file not found), return ProjectConfig::new_default()
         * If error (JSON parse), return Err with parse error
       - This allows graceful handling of new projects that haven't created settings.json yet

    6. Function: `load_project_state(project_path: &str) -> Result<ProjectState, String>`
       - Purpose: Load state from .maestro/state.json (or provide empty if not found)
       - Logic:
         * Try ProjectState::load_from_project(project_path)
         * If error (file not found), return ProjectState::empty()
         * If error (JSON parse), return Err with parse error

    7. Function: `ensure_maestro_folder_exists(project_path: &str) -> Result<(), String>`
       - Purpose: Safety check before any file operations
       - Logic:
         * Check if Path::new(project_path).join(".maestro") exists
         * If not, call create_project_maestro_folder()
         * Return Ok(()) if exists after check

    All functions should use Result<T, String> for Tauri IPC compatibility. Error messages should be descriptive with context (project path, operation name).

    Reference: 18-RESEARCH.md Pattern 1 and Common Pitfalls section (Pitfall 1 on Windows path handling).
