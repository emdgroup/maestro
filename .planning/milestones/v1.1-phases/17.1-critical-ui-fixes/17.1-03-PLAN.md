<task type="auto">
  <name>Add system accent color loading to ThemeProvider</name>
  <files>src/providers/ThemeProvider.tsx</files>
  <action>
Update src/providers/ThemeProvider.tsx to load and inject system accent color:

1. In the existing useEffect or create a new one:
   - Create function `loadSystemAccentColor()` that:
     - Gets current theme preference (isDark from matchMedia or from settings)
     - Determines accent color based on theme:
       - Light mode: "217 91% 35%" (blue, WCAG AA compliant)
       - Dark mode: "217 91% 71%" (lighter blue for dark mode readability)
     - Injects into CSS variable: `document.documentElement.style.setProperty("--accent", accentColor)`
     - Logs success to console: `[Theme] Accent color set to: ...`

2. Call this function:
   - On mount (useEffect with empty deps)
   - When theme preference changes (prefers-color-scheme media query listener)
   - Update deps array to include theme preference if using from settings

3. Error handling:
   - Wrap in try-catch
   - Log errors to console but don't throw (graceful fallback to CSS default)
   - Fallback accent color already in CSS, so missing injection just uses default

Code pattern:
```typescript
useEffect(() => {
  function loadSystemAccentColor() {
    try {
      const isDark = window.matchMedia("(prefers-color-scheme: dark)").matches;
      const accentColor = isDark ? "217 91% 71%" : "217 91% 35%";
      document.documentElement.style.setProperty("--accent", accentColor);
      console.log("[Theme] Accent color set to:", accentColor);
    } catch (err) {
      console.error("[Theme] Failed to set accent color:", err);
    }
  }

  loadSystemAccentColor();

  // Listen for system theme changes
  const mq = window.matchMedia("(prefers-color-scheme: dark)");
  mq.addEventListener("change", loadSystemAccentColor);
  return () => mq.removeEventListener("change", loadSystemAccentColor);
}, []);
```

Note: Tauri doesn't provide native system accent color API, so we use prefers-color-scheme as proxy. Real platform accent color (Windows accent, macOS accent, Linux XDG color) would require platform-specific calls beyond scope of v1.1.
