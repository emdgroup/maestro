# Phase 17.1 Plan 02: Modern Header with Project Dropdown and Tab Navigation Summary

**One-liner:** Modern AppHeader component with inline project dropdown, tab-based page navigation, and status indicator matching design patterns from exemple/ reference.

## Objective Completed

Redesigned the application header to match modern UX patterns from exemple/, featuring inline project dropdown and tab navigation for Tasks/Agents/Worktrees/Settings. This replaces the separate ProjectPicker screen with a compact, integrated header-based navigation.

## What Was Built

### 1. Modern AppHeader Component (src/components/AppHeader.tsx)

**Design:**
- **Left section (flex-shrink-0):** Logo "GSD Agent Orchestrator" + vertical divider + project Select dropdown
  - Logo in text-sm font-semibold tracking-tight
  - Divider: h-4 w-px bg-border
  - Project selector: h-7 w-[160px] bg-muted border-none with FolderOpen icon
- **Center section (flex-1):** Tab navigation with 4 buttons
  - Tasks (LayoutDashboard icon) → kanban view
  - Agents (Bot icon) → agents view
  - Worktrees (GitBranch icon) → worktrees view
  - Settings (Settings icon) → settings view
  - Active tab: bg-muted text-foreground
  - Inactive tab: text-muted-foreground with hover:bg-muted/50 transition
- **Right section (flex-shrink-0):** Status indicator + New Agent button
  - Animated pulse dot (bg-accent) with "{count} running" text
  - New Agent button: size="sm" h-7 gap-1.5 text-xs with Plus icon

**Technical details:**
- Component uses TypeScript with ViewType union ("kanban" | "agents" | "worktrees" | "settings")
- Props interface: currentProject, activeView, onViewChange, projects[], onProjectChange, agentCount
- useMemo hook for project options memoization to prevent unnecessary recalculations
- cn() utility from @/lib/utils for conditional className composition
- Imports: lucide-react icons, shadcn/ui Select components, Project type from bindings

### 2. App.tsx Integration

**Changes:**
- Added projects state to track available projects for dropdown
- Added loadAllProjects() async function calling get_projects IPC
- Updated activePage state type to union of view types with TypeScript strictness
- Modified initialization effect to load all projects on app mount
- Updated handleProjectSelected to reload projects list after selection
- Pass projects list to AppHeader.onProjectChange callback

**Routing logic:**
- After project selected, AppHeader renders with proper props
- Each tab click updates activePage state via onViewChange
- Main element conditionally renders:
  - activePage === "kanban" → KanbanBoard
  - activePage === "agents" → AgentMonitor
  - activePage === "worktrees" → WorktreeManager
  - activePage === "settings" → ProjectSettingsModal in container

**Props flowing to AppHeader:**
```typescript
<AppHeader
  currentProject={currentProject}           // Current Project object or null
  activeView={activePage}                   // Current page: "kanban" | "agents" | "worktrees" | "settings"
  onViewChange={setActivePage}              // Tab click handler
  projects={projects}                       // Project[] from state for dropdown
  onProjectChange={handleProjectSelected}   // Project dropdown change handler
  agentCount={0}                            // Agent count for status indicator (0 for now)
/>
```
