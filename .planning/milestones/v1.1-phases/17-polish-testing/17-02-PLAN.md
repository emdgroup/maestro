  <name>Task 1: Manual color contrast verification (WCAG AA 4.5:1)</name>
  <files>src/index.css, src/components/Button.tsx, src/components/Input.tsx, src/components/Card.tsx</files>
  <action>
    1. Extract all semantic color pairs from src/index.css and tailwind.config.ts:
       - Primary text (foreground) on background
       - Secondary text (muted-foreground) on background
       - Button text on button background
       - Link text (accent) on background
       - Disabled text on disabled background
       - Focus ring color on background
       - Status colors (success, warning, error) on backgrounds
    2. For each color pair, calculate WCAG AA contrast ratio:
       - Convert HSL to sRGB (using formula: https://www.w3.org/TR/WCAG21/#dfn-relative-luminance)
       - Calculate relative luminance for each color
       - Calculate contrast ratio: (lighter + 0.05) / (darker + 0.05)
       - Target: ≥ 4.5:1 for normal text
    3. Test in BOTH light and dark modes:
       - Light mode: dark text on light background
       - Dark mode: light text on dark background
       - Document ratio for each mode
    4. Verify using Chrome DevTools as secondary check:
       - Open app with `pnpm dev`
       - Inspect an element using a semantic color (e.g., button)
       - DevTools should show contrast ratio in color picker
       - Compare against calculated value
    5. Document findings:
       - Create spreadsheet-style table:
         | Color Pair | Light Mode Ratio | Dark Mode Ratio | Status |
         | Primary text | 12.5:1 | 10.2:1 | ✓ PASS |
         | Disabled text | 2.8:1 | 2.5:1 | ✗ FAIL |
       - Flag any < 4.5:1 for remediation
    6. If failures detected, add task to Phase 17-02 remediation (increase opacity/adjust colors)
  </action>
  <verify>
    - Contrast calculations documented for all semantic color pairs
    - Chrome DevTools confirms calculated ratios match
    - All text (normal and UI components) ≥ 4.5:1 in both modes
    - Disabled states checked (3:1 minimum for UI components acceptable, but prefer higher)
    - Table generated with pass/fail status
  </verify>
  <done>Color contrast audit complete, all text meets 4.5:1 WCAG AA requirement in both light and dark modes, any failures documented for remediation</done>
</task>
