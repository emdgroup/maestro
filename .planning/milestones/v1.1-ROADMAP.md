# Milestone v1.1: UI/UX Polish

**Status:** ✅ SHIPPED 2026-03-16
**Phases:** 13-22 (including 17.1 insertion)
**Total Plans:** 36

## Overview

This milestone fixes critical v1.0 bugs (mock IPC leak, Rust warnings) then transforms the UI from functional to beautiful. Establishes a modern CSS framework (Tailwind CSS 4.1 + shadcn/ui), implements a complete theming system with no flash-on-startup, migrates all components to a design system, redesigns all five major pages (Kanban, Agent Monitor, Worktree Manager, Settings, Header) matching the clean, modern mockup aesthetic with system-first theme and OS accent colors. Also includes Maestro rebranding, frontend architecture overhaul (views/services/TanStack Query), and transparent stale project cleanup.

## Milestone Summary

**Decimal Phases:**

- Phase 17.1: Critical UI Fixes (inserted after Phase 17 for urgent UX/production bug fixes)

**Key Decisions:**

- Tailwind CSS 4.1 + @tailwindcss/vite: Official recommendation, 8kB bundle savings, native Vite integration
- shadcn/ui: Copy-paste workflow reduces coupling, theme-aware via CSS variables
- System-first theme: Follows OS theme preference, respects user's system settings
- ThemeProvider via React Context API with system theme detection and dual preload (frontend + Tauri)
- Project-local .maestro/ folder for state and settings per project (not global database)
- Maestro rebranding (from "GSD Orchestrator") — technical GSD references kept for consistency
- views/services/components architecture: standard industry structure for maintainability
- TanStack Query for all IPC: 37 hooks across 5 services, 0 direct invoke() calls in UI layer
- DB mutex released before async SSH I/O in get_connection_projects (avoids blocking)

**Issues Resolved:**

- Production mode folder selection stuck at picker (Phase 17.1)
- System accent color not applied dynamically (Phase 17.1)
- Mock IPC leak into production bundles (Phase 13)
- Rust build warnings (Phase 13)
- E0597 Rust lifetime error in get_connection_projects scoped block pattern (Phase 22)

**Issues Deferred:**

- Light mode full implementation (dark-first, system theme respected)
- Full accessibility audit for v1.1 additions
- Custom theme color picker
