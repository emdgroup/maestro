<task type="auto">
  <name>Task 2: Create TaskSettingsModal with override fields and null handling</name>
  <files>src/components/TaskSettingsModal.tsx, src/components/TaskSettingsModal.css</files>
  <action>
    Create TaskSettingsModal.tsx with react-hook-form + Radix UI Dialog (similar to ProjectSettingsModal but for task-level overrides).

    Props interface:
    - isOpen: boolean
    - onClose: () => void
    - task: Task
    - projectId: number

    Component logic:
    - Import AVAILABLE_MCP_SERVERS, AVAILABLE_SKILLS, AVAILABLE_MODELS from configStore (created in Plan 02)
    - Use useForm<TaskSettingsFormData>() hook
    - formData structure: { model: string, mcp_allowlist: Record<string, boolean>, skills_override: Record<string, boolean> }

    Key design: Null vs Vec distinction (from CONTEXT.md)
    - task.model_override: null means "use project default", non-null means "override"
    - task.mcp_allowlist: null means "use project defaults", empty vec [] means "no MCP servers", non-empty vec means "these servers only"
    - task.skills_override: null means "use project defaults", non-empty vec means "these skills only"

    On mount:
    - Populate form defaultValues from task override fields with null handling:
      * If model_override is null: set form value to "Use Project Default" (empty string or special value)
      * If model_override is non-null: set form value to that model
      * If mcp_allowlist is null: all checkboxes unchecked (no override)
      * If mcp_allowlist is vec: checkboxes checked for items in vec
      * Same pattern for skills_override
    - Helper function parseNullableVecToCheckboxRecord(vec: string[] | null) to convert array to checkbox record with "checked" for items in array

    Form fields (all optional - leaving empty means task uses project defaults):
    1. Model Override: <select> with options:
       - "Use Project Default" (empty value or null)
       - Each AVAILABLE_MODEL value
       On change: if empty, send null; if selected, send model string
    2. MCP Allowlist: Checkboxes for each AVAILABLE_MCP_SERVERS item
       - No checkboxes checked: send null (use project defaults)
       - Some checked: send vec of checked items (override to those only)
    3. Skills Override: Checkboxes for each AVAILABLE_SKILLS item
       - No checkboxes checked: send null (use project defaults)
       - Some checked: send vec of checked items (override to those only)

    Key design note (from CONTEXT.md): Task settings are FULL REPLACEMENT, not additive. If user checks any MCP servers, those become the ONLY servers for this task (project defaults ignored). No inheritance display - form shows only task overrides.

    Submit handler (onSubmit):
    - Convert form data back to send format:
      * model_override: if form value is empty, send null; else send selected model string
      * mcp_allowlist: if no checkboxes checked, send null; else send vec of checked items
      * skills_override: if no checkboxes checked, send null; else send vec of checked items
    - Call invoke('update_task_settings', { task_id: task.id, model_override, mcp_allowlist, skills_override }) with await
    - On success: close modal via onClose()
    - On error: display error message in modal banner, do NOT close modal

    Radix Dialog structure (similar to ProjectSettingsModal):
    - <Dialog.Root open={isOpen} onOpenChange={onClose}>
    - <Dialog.Title>Task Configuration Overrides</Dialog.Title>
    - <Dialog.Description> explaining override behavior and null semantics
    - Form with three fieldsets
    - Error banner if save fails

    Reference: React Hook Form docs + ProjectSettingsModal pattern from Plan 02. Import AVAILABLE_* constants from configStore.
  </action>
