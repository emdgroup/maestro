  <name>Task 3: Implement IPC handlers for configuration CRUD</name>
  <files>src-tauri/src/ipc/handlers.rs</files>
  <action>
    Add three new Tauri IPC command handlers to handlers.rs:

    1. get_project_settings(state: State<Arc<AppState>>, project_id: i32) -> Result<ProjectConfigResponse, String>
       - Lock database connection
       - Query settings table for keys: model_default, mcp_allowlist, skills_default
       - Deserialize JSON arrays from database
       - Return struct with model_default: String, mcp_allowlist: Vec<String>, skills_default: Vec<String>
       - If keys missing, return defaults (model: "claude-opus-4-5", mcp: [], skills: [])

    2. update_project_settings(state: State<Arc<AppState>>, project_id: i32, settings: ProjectConfigRequest) -> Result<(), String>
       - settings contains: model_default: String, mcp_allowlist: Vec<String>, skills_default: Vec<String>
       - Lock database connection, start transaction
       - Upsert settings table: model_default (plain string), mcp_allowlist (JSON), skills_default (JSON)
       - Commit transaction
       - Return Result::Ok on success

    3. update_task_settings(state: State<Arc<AppState>>, task_id: i32, settings: TaskConfigRequest) -> Result<(), String>
       - settings contains: model_override: Option<String>, mcp_allowlist: Option<Vec<String>>, skills_override: Option<Vec<String>>
       - Lock database connection
       - UPDATE tasks SET model_override=?, mcp_allowlist=?, skills_override=? WHERE id=?
       - Serialize None as NULL, Some(vec) as JSON array string
       - Return Result::Ok on success

    Add #[tauri::command] attribute to each handler.
    Reference Phase 02-02 pattern for skills JSON serialization (serde_json::to_string/from_str).

    Handlers must NOT check inheritance - just read/write raw task fields. Inheritance resolved on agent execution layer (future phase).
  </action>

    Verify handlers compile without errors. No SQL syntax errors in UPDATE/INSERT statements.
  </verify>
  <done>

  </done>
