       ```rust
       #[tauri::command]
       pub async fn sync_github_issues(
           state: State<'_, Arc<AppState>>,
           project_id: i32,
           owner: String,
           repo: String,
           token: String,
       ) -> Result<SyncResult, String> {
           // Use reqwest to call GitHub API (https://api.github.com/repos/{owner}/{repo}/issues)
           // Set Authorization: Bearer {token} header
           // Fetch open issues (status: "open" in query)
           // For each issue:
           //   - external_id = issue.number.to_string()
           //   - Check if task with external_id exists:
           //     * If exists: UPDATE name = issue.title, description = issue.body, keep status unchanged
           //     * If new: INSERT with status='backlog', is_imported=true, import_source='github'
           // Return SyncResult { imported_count, updated_count, error_message }
           // On error: Return SyncResult with error_message set (non-fatal)
       }
       ```
    2. Implementation steps:
       - Construct GitHub API URL: https://api.github.com/repos/{owner}/{repo}/issues?state=open&per_page=100
       - Use reqwest client (async) to fetch JSON
       - Parse response as Vec<GitHubIssue> (deserialize)
       - For each issue:
         * Query tasks table: SELECT id, status FROM tasks WHERE external_id = issue.number
         * If found: UPDATE tasks SET name = issue.title, description = issue.body WHERE id = ?
         * If not found: INSERT INTO tasks (..., is_imported=true, import_source='github', external_id=issue.number, ...)
       - Wrap in transaction to ensure atomicity

    Note: GitHub API docs: https://docs.github.com/en/rest/issues. Pagination handled by per_page=100 (adjust if needed).
  </action>

       ```rust
           //     * If exists: UPDATE name = issue.fields.summary, description = issue.fields.description, keep status

           // Return SyncResult { imported_count, updated_count, error_message }
       }
       ```

       - Construct Jira API URL: https://{host}/rest/api/3/search
       - Encode auth: base64(email:api_token)

       - For each issue:
         * Query tasks: SELECT id, status FROM tasks WHERE external_id = issue.key
         * If found: UPDATE tasks SET name = issue.fields.summary, description = issue.fields.description WHERE id = ?
         * If not found: INSERT into tasks (..., is_imported=true, import_source='jira', external_id=issue.key, ...)
       - Handle Jira error responses (401 Unauthorized, 403 Forbidden, etc.)

    Note: Jira Cloud API docs: https://developer.atlassian.com/cloud/jira/rest/v3/. JQL examples: "project = PROJ", "status = 'To Do'".
