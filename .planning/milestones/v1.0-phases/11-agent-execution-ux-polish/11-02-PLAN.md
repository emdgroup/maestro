  <action>
Update lease_worktree function to handle pool exhaustion with automatic retry.

Current lease_worktree implementation (Phase 3-02) leases an available worktree from pool. Phase 11 adds retry logic for pool exhaustion:

1. If lease_worktree finds no Available worktrees:
   - Check if any worktrees are Dirty or InUse (temporary states)
   - Retry up to 3 times with 500ms exponential backoff (500ms, 1s, 1.5s)
   - On each retry, check again for Available worktrees
   - If still no Available after retries, attempt to create a new worktree (pool expansion)

2. If pool expansion needed:
   - Call create_worktree to add new worktree to pool
   - Immediately lease the newly created worktree
   - Return the leased worktree (no error to caller)

3. If all retries fail AND pool expansion fails:
   - Return Err("Failed to lease or create worktree: pool exhausted and creation failed")
   - Frontend will show error toast

Implementation approach:
   - Use tokio::time::sleep for backoff delays (async safe)
   - Wrap lease_worktree logic in a retry loop (3 iterations max)
   - After each failed lease attempt, wait and retry
   - On final failure, attempt create_worktree from Phase 3-04

Reference Phase 3-02 SUMMARY (lease_worktree implementation) and Phase 3-04 (create_worktree).

CRITICAL: Use async sleep (tokio::time::sleep), not std::thread::sleep. Handler is async, blocking sleep will freeze IPC.

CRITICAL: Exponential backoff: 500ms * 2^attempt (0, 1, 2) = 500ms, 1s, 1.5s. Total max wait: 3s before fatal error.

  <action>
Modify spawn_agent_execution handler to lease worktree before spawning agent CLI.

Current flow (Phase 4-02): spawn_agent_execution creates execution log and spawns agent with placeholder worktree path.

New flow (Phase 11):
1. Lease worktree from pool (automatic, with retry)
2. Create execution log
3. Spawn agent with actual worktree path
4. On completion, return worktree to pool

Implementation in spawn_agent_execution handler:

```rust
#[tauri::command]
pub async fn spawn_agent_execution(
  state: State<Arc<AppState>>,
  project_id: i32,
  task_id: i32,
  repo_path: String,
) -> Result<i32, String> {
  // Phase 11: Lease worktree BEFORE spawning agent
  let conn = state.db.lock().map_err(|e| format!("Failed to lock DB: {}", e))?;
  let worktree = lease_worktree(&conn, project_id, task_id, &repo_path).await?;
  let worktree_id = worktree.id;
  let worktree_path = worktree.path.clone();
  drop(conn); // Release lock before async operations

  // Create execution log (existing code)
  let conn = state.db.lock().map_err(|e| format!("Failed to lock DB: {}", e))?;
  let exec_log = create_execution_log(&conn, task_id)?;
  let exec_id = exec_log.id;
  drop(conn);

  // Spawn agent in background with actual worktree path
  tokio::spawn(async move {
    let result = run_agent_process(&worktree_path, task_id).await;

    // Update execution log (existing code)
    let conn = state.db.lock().unwrap();
    update_execution_log(&conn, exec_id, result).ok();

    // Return worktree to pool (existing code)
    return_worktree(&conn, worktree_id).ok();
  });

  Ok(exec_log.id)
}
```

Key changes:
- Add `lease_worktree` call before `tokio::spawn`
- Pass actual `worktree.path` to `run_agent_process` (not placeholder)
- Store `worktree_id` for return_worktree call
- On lease failure, return Err immediately (frontend catches and shows toast)

CRITICAL: Lock must be held during lease_worktree AND create_execution_log, but released before tokio::spawn. Database connection can't be shared across await boundary.

CRITICAL: Error from lease_worktree propagates to frontend as string. Frontend's executeTask() catch block will show error toast (existing error handling in TaskCard.tsx lines 50-51).
