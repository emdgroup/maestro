   ```rust
   pub struct CircularBuffer {
       max_lines: usize,
       lines: VecDeque<String>,
   }

   impl CircularBuffer {
       pub fn new(max_lines: usize) -> Self { ... }
       pub fn append(&mut self, line: String) { 
           if self.lines.len() >= self.max_lines {
               self.lines.pop_front();
           }
           self.lines.push_back(line);
       }
       pub fn get_all(&self) -> String {
           self.lines.iter().map(|l| l.as_str()).collect::<Vec<_>>().join("\n")
       }
   }
   ```

  <name>Task 2: Implement append_terminal_output handler to persist PTY output to database periodically</name>
  <files>src-tauri/src/ipc/handlers.rs</files>
  <action>
1. Add new IPC handler append_terminal_output:
   ```rust
   #[tauri::command]
   pub async fn append_terminal_output(
       state: State<'_, Arc<AppState>>,
       task_id: i32,
       output: String,
   ) -> Result<(), String> {
       // Get database connection
       let db = state.db.lock().map_err(|e| e.to_string())?;
       
       // UPDATE execution_logs SET terminal_output = terminal_output || output 
       //   WHERE task_id = ? AND status IN ('running', 'failed', 'complete')
       // (Append to existing output)
       
       db.execute(
           "UPDATE execution_logs SET terminal_output = COALESCE(terminal_output, '') || ? 
            WHERE task_id = ? AND status IN ('running', 'failed', 'complete')
            ORDER BY id DESC LIMIT 1",
           rusqlite::params![&output, task_id],
       ).map_err(|e| e.to_string())?;
       
       Ok(())
   }
   ```

2. Update attach_terminal handler in pty.rs:
   - In the streaming loop (after PTY reader gets output):
   - Call invoke('append_terminal_output', { taskId, output: chunk }) periodically
   - Use `tokio::time::interval(Duration::from_millis(500))` to batch appends (avoid too many DB writes)
   - Or append on larger chunks (e.g., >4KB accumulated)

3. Register handler in main.rs generate_handler macro

Reference: RESEARCH.md "Integration with Existing Code" mentions this buffering pattern for history.

Key: Do NOT call append_terminal_output on every single read - batch them to reduce DB contention. Use interval for periodic flushing.
