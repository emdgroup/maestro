**1. Fix non-standard SQL in execution_handlers.rs:**

In `src-tauri/src/ipc/execution_handlers.rs`, in the `append_terminal_output` function (around lines 608-615), replace the UPDATE statement:

```rust
// BEFORE (non-standard ORDER BY in UPDATE):
"UPDATE execution_logs
 SET terminal_output = COALESCE(terminal_output, '') || ?
 WHERE task_id = ? AND status IN ('running', 'failed', 'complete')
 ORDER BY id DESC LIMIT 1"
```

With the safe subquery form:

```rust
// AFTER (standard SQL with subquery):
"UPDATE execution_logs
 SET terminal_output = COALESCE(terminal_output, '') || ?1
 WHERE id = (
     SELECT id FROM execution_logs
     WHERE task_id = ?2 AND status IN ('running', 'failed', 'complete')
     ORDER BY id DESC LIMIT 1
 )"
```

Note the positional parameters change from `?` to `?1` and `?2`. The `rusqlite::params![&output, task_id]` call stays the same.

Also remove or update the comment block (lines 617-623) that says "The ORDER BY in an UPDATE is non-standard..." since it no longer applies. Replace with a brief comment: `// Subquery targets the most recent execution log for this task`.

**2. Fix WorktreeManager diff loading condition:**

In `src/components/execution/WorktreeManager.tsx`, in the diff body section (around line 300), the current condition is:

```tsx
{selectedWorktree.git_status === "" && !diffLoading ? (
  <div className="...">No uncommitted changes</div>
```

The problem: when `git_status` is non-empty but diff is still loading, `diffFiles.length` is 0 so we fall through to the DiffViewer with `loading={diffLoading}` which shows "No changes to display" momentarily before loading takes effect.

Fix by checking `diffLoading` first. Replace the entire conditional block with:

```tsx
  <div className="p-4 space-y-4">
    {diffFiles.map((file, i) => (
      <DiffViewer key={i} diffFile={file} loading={false} />
    ))}
  </div>
```
