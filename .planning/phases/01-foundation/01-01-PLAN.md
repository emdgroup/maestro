<task type="auto">
  <name>Task 2: Create database schema module with PRAGMA versioning</name>
  <files>src-tauri/src/db/schema.rs</files>
  <action>
1. Create directory: `mkdir -p src-tauri/src/db`

2. Create src-tauri/src/db/schema.rs with:
   - Constant SCHEMA_VERSION = 1
   - Constant SCHEMA_V1 containing SQL DDL for:
     * projects table (id, name, path, created_at, unique constraint on name)
     * tasks table (id, project_id FK, name, description, status, created_at, updated_at)
     * worktrees table (id, project_id FK, branch_name, path, status, leased_at, returned_at)
     * execution_logs table (id, task_id FK, output, status, started_at, completed_at)
     * settings table (key TEXT PRIMARY KEY, value TEXT, updated_at TEXT) for app configuration (CFG-01 requirement)
   - Function initialize_schema(conn: &Connection) -> Result<(), rusqlite::Error>:
     * Read PRAGMA user_version
     * If current < SCHEMA_VERSION, execute SCHEMA_V1
     * Update PRAGMA user_version to SCHEMA_VERSION
   - Enable foreign keys: PRAGMA foreign_keys = ON

3. Design rationale:
   - Use INTEGER PRIMARY KEY (no AUTOINCREMENT — standard for MVP)
   - Timestamps as TEXT (ISO 8601) for JSON serialization simplicity
   - status columns as TEXT (enum strings, not numbers)
   - All FK constraints active via PRAGMA
  </action>
