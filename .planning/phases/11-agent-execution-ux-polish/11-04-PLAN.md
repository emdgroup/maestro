Update ExecutionHistory.tsx to detect new failures and show toast notifications.

Current polling (lines 50+) fetches execution logs every 5 seconds. Extend to detect failures:

1. Track previous state with useRef:
   ```typescript
   const previousLogsRef = useRef<ExecutionLog[]>([]);
   ```

2. In loadExecutionLogs function, after fetching logs:
   ```typescript
   const loadExecutionLogs = async () => {
     try {
       const logs = await invoke<ExecutionLog[]>('get_execution_logs', { task_id: taskId });
       const previousLogs = previousLogsRef.current;

       // Detect NEW failures (logs that are Failed now but weren't Failed before)
       const newFailedLogs = logs.filter(log =>
         log.status === 'Failed' &&
         !previousLogs.find(p => p.id === log.id && p.status === 'Failed')
       );

       // Show toast for each new failure
       newFailedLogs.forEach(log => {
         const errorType = log.error_event?.error_type || 'Unknown Error';
         const message = `Failed: ${taskName} — ${errorType}`;
         toast.error(message, { duration: 10000 }); // 10s auto-dismiss
       });

       // Update refs and state
       previousLogsRef.current = logs;
       setLogs(logs);
     } catch (error) {
       console.error('Failed to load execution logs:', error);
     }
   };
   ```

3. Initialize previousLogsRef on mount:
   ```typescript
   useEffect(() => {
     loadExecutionLogs();
     const interval = setInterval(loadExecutionLogs, 5000);
     return () => clearInterval(interval);
   }, [taskId]);
   ```

Reference RESEARCH.md Example 3 (lines 399-432) for failure toast pattern.

CRITICAL: Compare log.id to detect NEW failures (not repeated). If same log ID appears with status='Failed' in both previous and current, don't show toast again.

CRITICAL: Message format must be exactly: `Failed: ${taskName} — ${errorType}`. Never include full error.message or suggestions in toast (that's for modal detail view).
