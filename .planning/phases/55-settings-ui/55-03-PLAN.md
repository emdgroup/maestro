**2. Wire cascade check into MainLayout (or equivalent project wrapper):**

Find the component that renders AFTER a project is opened (where `projectId` is available and the main UI tabs render). This is likely `src/views/MainLayout.tsx` or a similar wrapper.

Add the cascade check:
```tsx
import { useListIntegrations, useProjectTicketingConfig } from "@/services/integration.service";
import { IntegrationMissingDialog } from "@/components/project-picker/IntegrationMissingDialog";

// Inside the component:
const { data: integrations, isLoading: integrationsLoading } = useListIntegrations();
const { data: ticketingConfig, isLoading: ticketingLoading } = useProjectTicketingConfig(projectId);

const [showMissingDialog, setShowMissingDialog] = useState(false);
const [missingProvider, setMissingProvider] = useState<string | null>(null);

useEffect(() => {
  if (integrationsLoading || ticketingLoading) return;
  if (!ticketingConfig) {
    setShowMissingDialog(false);
    return;
  }
  // Check if the configured provider's integration still exists and is connected
  const integration = integrations?.find(i => i.provider === ticketingConfig.provider);
  if (!integration || !integration.connected) {
    setMissingProvider(ticketingConfig.provider);
    setShowMissingDialog(true);
  } else {
    setShowMissingDialog(false);
  }
}, [integrations, ticketingConfig, integrationsLoading, ticketingLoading]);
```

Render the dialog:
```tsx
<IntegrationMissingDialog
  open={showMissingDialog}
  projectId={projectId}
  provider={missingProvider ?? ""}
  onFixIntegration={() => {
    // Navigate back to project picker / connection screen
    // Use the existing navigation mechanism (e.g., useProjectNavigation or similar)
  }}
  onDropConfig={() => {
    setShowMissingDialog(false);
  }}
/>
```

When `showMissingDialog` is true, the main content should still render behind it (the dialog is a modal overlay). Alternatively, if the project wrapper supports conditional rendering, block the main UI until resolved. The simplest approach: just render the Dialog as an overlay — it's blocking because it has no dismiss.
