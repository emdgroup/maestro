**1. Create `src-tauri/src/ipc/integration_handlers.rs`** with 4 IPC handlers:

```rust
use std::sync::Arc;
use tauri::State;
use crate::db::AppState;
use crate::models::integration::{IntegrationStatus, IntegrationCredentials, CredentialSource};
use crate::models::project_config::now_rfc3339;
use crate::ticketing::keychain::{KeychainStore, KeychainOutcome};

const KNOWN_PROVIDERS: &[&str] = &[
    "github", "gitlab", "forgejo", "linear", "jira_cloud", "azuredevops",
];
```

**`list_integrations`** (per D-03 — probes all 7 keys):
- Iterates KNOWN_PROVIDERS
- For each, calls `KeychainStore::get_integration(provider, &app_state.app_data_dir)`
- On `KeychainOutcome::Keychain(Some(creds))` or `FileFallback(Some(creds))`: push `IntegrationStatus { provider, connected: true, display_name: creds.display_name, source: Some(creds.source) }`
- On `None` or error: push `IntegrationStatus { provider, connected: false, display_name: None, source: None }`
- Special case for "github": also probe gh CLI via `crate::ticketing::github::try_gh_cli_token().await`. If keyring returns None but gh CLI returns Some(token), return connected=true with source=GhCli, display_name=None. Do NOT store the gh CLI token into keyring — it's ephemeral and re-probed each call (per D-18 and RESEARCH.md Pitfall 3).
- Returns `Result<Vec<IntegrationStatus>, String>`

**`save_integration`** (per D-02 — stores credentials globally):
- Signature: `provider: String, token: String, instance_url: Option<String>, email: Option<String>`
- Validates provider is in KNOWN_PROVIDERS, else return Err
- Calls `validate_credentials(&provider, &token, instance_url.as_deref(), email.as_deref()).await?` to verify credentials against provider API (returns display_name String)
- Constructs `IntegrationCredentials { token, instance_url, email, display_name: Some(display_name), connected_at: now_rfc3339(), source: CredentialSource::Manual }`
- Calls `KeychainStore::store_integration(&provider, &creds, &app_state.app_data_dir)?`
- If outcome is `FileFallback`, emit keyring-unavailable warning via `app_state.app_handle.emit("ticketing:keyring-unavailable", ())`
- Returns `Result<String, String>` (the display_name)

**`delete_integration`** (per D-03):
- Signature: `provider: String`
- Calls `KeychainStore::delete_integration(&provider, &app_state.app_data_dir)?`
- Returns `Result<(), String>`

**`test_integration`** (validation-only, no store):
- Signature: `provider: String, token: String, instance_url: Option<String>, email: Option<String>`
- Calls `validate_credentials(&provider, &token, instance_url.as_deref(), email.as_deref()).await?`
- Returns `Result<String, String>` (the display_name)

**`validate_credentials` helper** (private async fn in same file):
- Match on provider string:
  - `"github"`: GET `https://api.github.com/user` with `Authorization: Bearer {token}`, `User-Agent: maestro/1.0`. Parse login field. Return login.
  - `"gitlab"`: GET `{instance_url}/api/v4/user` with `PRIVATE-TOKEN: {token}`. Parse username field. Return username.
  - `"forgejo"`: GET `{instance_url}/api/v1/user` with `Authorization: token {token}`. Parse login field. Return login.
  - `"linear"`: POST `https://api.linear.app/graphql` with `Authorization: {token}`, body `{"query":"{ viewer { id name } }"}`. Parse data.viewer.name. Return name.
  - `"jira_cloud"`: GET `{instance_url}/rest/api/3/myself` with Basic auth (base64 of `{email}:{token}`). Parse displayName field. Return displayName.
  - `"azuredevops"`: GET `{instance_url}/_apis/connectionData?api-version=7.1` with Basic auth (base64 of `:{token}`). Parse authenticatedUser.providerDisplayName. Return providerDisplayName.
- All HTTP requests use `reqwest::Client::builder().timeout(Duration::from_secs(15)).build()`
- On 401: return `Err(format!("{}: bad credentials", provider_display_name))`
- On non-success: return `Err(format!("{}: API error {}", provider_display_name, status))`
- Where `instance_url` is needed but None: return `Err(format!("{}: instance_url required", provider))`
- Where `email` is needed but None: return `Err(format!("{}: email required", provider))`
- Use `crate::ticketing::normalize_instance_url()` on any instance_url/site_url/base_url/org_url before making requests.
