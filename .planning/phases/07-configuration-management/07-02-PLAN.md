  <files>src/components/ProjectSettingsModal.tsx, src/components/ProjectSettingsModal.css</files>
  <action>
    Create ProjectSettingsModal.tsx (React component using react-hook-form + Radix UI Dialog).

    Props interface:
    - isOpen: boolean
    - onClose: () => void
    - projectId: number

    Component logic:
    - Use useForm<ProjectSettingsFormData>() hook with react-hook-form
    - formData structure: { model_default: string, mcp_servers: Record<string, boolean>, skills: Record<string, boolean> }
    - Fetch initial settings via invoke('get_project_settings', { project_id: projectId }) on mount (useEffect)
    - Populate form with fetched values using defaultValues in useForm
    - Parse checkbox groups: convert Vec<String> from backend into Record<string, boolean> for form (e.g., {"filesystem": true, "web": false})

    Form fields:
    1. Model Dropdown: <select> using register("model_default"), options from AVAILABLE_MODELS
    2. MCP Servers fieldset: Loop over AVAILABLE_MCP_SERVERS, checkbox input per server using register(`mcp_servers.${server}`)
    3. Skills fieldset: Loop over AVAILABLE_SKILLS, checkbox input per skill using register(`skills.${skill}`)

    Submit handler (onSubmit):
    - Convert checkbox records back to arrays: Object.entries(data.mcp_servers).filter(([_, enabled]) => enabled).map(([server]) => server)
    - Call invoke('update_project_settings', { project_id, settings: { model_default, mcp_allowlist, skills_default } }) with await
    - Catch errors and display in error state
    - Close modal on success

    Radix Dialog wrapper structure (use existing TaskModal.tsx as template):
    - <Dialog.Root open={isOpen} onOpenChange={onClose}>
    - <Dialog.Portal> with <Dialog.Overlay> and <Dialog.Content>
    - <Dialog.Title>Project Configuration</Dialog.Title>
    - <Dialog.Description> explaining what this modal does
    - Form with fieldsets for each section
    - Save and Close buttons
    - Error banner (if error state exists)

    Reference: Phase 02-03 TaskModal.tsx and react-hook-form docs for field registration pattern.
