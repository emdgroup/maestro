  <action>
    Create ProjectSettingsModal.tsx (React component using react-hook-form + Radix UI Dialog + configStore).

    Props interface:
    - isOpen: boolean
    - onClose: () => void
    - projectId: number

    Component logic:
    - Import useConfigStore and destructure: { model_default, mcp_allowlist, skills_default, isLoading, setState, setLoading, setError, clearError } = useConfigStore()
    - Use useForm<ProjectSettingsFormData>() hook with react-hook-form
    - formData structure: { model_default: string, mcp_servers: Record<string, boolean>, skills: Record<string, boolean> }
    - On modal open (useEffect with [isOpen] dependency):
      * setLoading(true), clearError()
      * Fetch via invoke('get_project_settings', { project_id: projectId })
      * setState() with fetched values to update Zustand store
      * setLoading(false)
      * Parse checkbox groups: convert Vec<String> from backend into Record<string, boolean> for form (e.g., {"filesystem": true, "web": false})
      * Populate form defaultValues with converted data
    - On form mount, use Zustand state values as defaultValues in useForm

    Form fields:
    1. Model Dropdown: <select> using register("model_default"), options from AVAILABLE_MODELS
    2. MCP Servers fieldset: Loop over AVAILABLE_MCP_SERVERS (from store export), checkbox input per server using register(`mcp_servers.${server}`)
    3. Skills fieldset: Loop over AVAILABLE_SKILLS (from store export), checkbox input per skill using register(`skills.${skill}`)

    Submit handler (onSubmit):
    - Convert checkbox records back to arrays: Object.entries(data.mcp_servers).filter(([_, enabled]) => enabled).map(([server]) => server)
    - Call invoke('update_project_settings', { project_id, settings: { model_default, mcp_allowlist, skills_default } }) with await
    - On success: setState() to update Zustand store with new values, close modal
    - On error: setError() to display error message in modal

    Radix Dialog wrapper structure (use existing TaskModal.tsx as template):
    - <Dialog.Root open={isOpen} onOpenChange={onClose}>
    - <Dialog.Portal> with <Dialog.Overlay> and <Dialog.Content>
    - <Dialog.Title>Project Configuration</Dialog.Title>
    - <Dialog.Description> explaining what this modal does
    - Form with fieldsets for each section
    - Save and Close buttons (disable Save button while isLoading)
    - Error banner (if error state exists)

    Reference: Phase 02-03 TaskModal.tsx, react-hook-form docs, and boardStore pattern for Zustand integration.
