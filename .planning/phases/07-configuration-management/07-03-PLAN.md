<task type="auto">
  <name>Task 2: Create TaskSettingsModal with override fields</name>
  <files>src/components/TaskSettingsModal.tsx, src/components/TaskSettingsModal.css</files>
  <action>
    Create TaskSettingsModal.tsx with react-hook-form + Radix UI Dialog (similar to ProjectSettingsModal but for task-level overrides).

    Props interface:
    - isOpen: boolean
    - onClose: () => void
    - task: Task
    - projectId: number

    Component logic:
    - Use useForm<TaskSettingsFormData>() hook
    - formData structure: { model: string, mcp_allowlist: Record<string, boolean>, skills_override: Record<string, boolean> }
    - On mount, fetch current task data (already have task prop) and get project settings for reference
    - Populate form defaultValues from task override fields (if null/undefined, show as unchecked for checkboxes, empty for model)
    - Helper function to convert task.mcp_allowlist (Vec<String> or null) to checkbox record
    - Helper function to convert task.skills_override (Vec<String> or null) to checkbox record

    Form fields (all optional - leaving empty means task uses project defaults):
    1. Model Override: <select> with option "Use project default" (empty value) + models from AVAILABLE_MODELS
    2. MCP Allowlist: Checkboxes for each MCP server (replaces project allowlist if any checked)
    3. Skills Override: Checkboxes for each skill (replaces project skills if any checked)

    Key design note (from CONTEXT.md): Task settings are FULL REPLACEMENT, not additive. If user checks any MCP servers, those become the ONLY servers for this task (project defaults ignored). No inheritance display - form shows only task overrides.

    Submit handler (onSubmit):
    - Convert form data back to send format: model_override (null if empty), mcp_allowlist (null or vec), skills_override (null or vec)
    - Call invoke('update_task_settings', { task_id: task.id, model_override, mcp_allowlist, skills_override })
    - Close modal on success

    Radix Dialog structure (similar to ProjectSettingsModal):
    - <Dialog.Root open={isOpen} onOpenChange={onClose}>
    - <Dialog.Title>Task Configuration Overrides</Dialog.Title>
    - <Dialog.Description> explaining override behavior
    - Form with three fieldsets

    Reference: React Hook Form docs + ProjectSettingsModal pattern from Plan 02.
  </action>
