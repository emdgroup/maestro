   The handler must:
   a. Search ACP sessions: lock `app_state.acp.sessions`, find entry where `proc.task_id == Some(task_id)`, extract the `log_id` key. Release lock immediately (scoped block).

   c. If both are None, return `Err(format!("No active session for task {}", task_id))`.

   f. AFTER all async work is done, acquire the sync DB mutex in its own block: UPDATE tasks SET status = 'Backlog', updated_at = ? WHERE id = ?. Release DB lock by ending the block.
   g. Emit two events: `app_state.app_handle.emit("tasks-changed", ()).ok()` and `app_state.app_handle.emit("sessions-changed", ()).ok()`.

   Critical constraint: The sync `app_state.db.lock()` must NEVER be held across an await point. All tokio async locks must be acquired and released before the sync DB lock.
