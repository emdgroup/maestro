**Step 3: Gate local-only parallel git status/diff (lines 85-115) on is_remote.**

For SSH projects, skip the parallel `tokio::spawn` of `get_worktree_status_local` and `git diff --shortstat` with `.current_dir()`. Instead, for remote projects, run `git status --porcelain` and `git diff --shortstat` via SSH for each worktree path. However, to keep scope manageable, for SSH projects populate `git_info` with empty defaults (empty string status, None diff_stat). The worktree list and DB cross-reference are the critical fixes. Detailed per-worktree SSH status can be added later.

Wrap the existing parallel block in `if !is_remote { ... } else { /* empty git_info */ }`:
```rust
let mut git_info: HashMap<String, (String, Option<String>)> = HashMap::new();
if !is_remote {
    // existing tokio::spawn parallel block for local git status + diff
    let handles: Vec<_> = disk_worktrees.iter().map(|wt| { /* existing code */ }).collect();
    for handle in handles {
        if let Ok((path, status, diff_stat)) = handle.await {
            git_info.insert(path, (status, diff_stat));
        }
    }
}
// For remote projects, git_info stays empty — worktrees show without status details
```

The DB query at lines 43-73 works for both local and remote since it queries by `project_id`. The result building at lines 122-159 uses `git_info.get(&wt.path).cloned().unwrap_or_default()` which gracefully returns empty status when git_info has no entry.
