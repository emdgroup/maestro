**1. AcpProcess struct:**
```rust
use tokio::process::{Child, ChildStdin};
use tokio::io::BufWriter;
use tokio::sync::oneshot;

pub struct AcpProcess {
    /// Child process handle — kill_on_drop(true) ensures cleanup
    pub child: Child,
    /// Write half of piped stdin — used to send SpawnRequest, PromptRequest, PermitResponse
    pub stdin_writer: BufWriter<ChildStdin>,
    /// Cancel signal for the background reader task (send () to stop the reader loop)
    pub reader_cancel_tx: Option<oneshot::Sender<()>>,
}
```

Use `Option<oneshot::Sender<()>>` for `reader_cancel_tx` so it can be `.take()`-ed during cleanup (oneshot::Sender is not Clone).

**2. spawn_acp_process function:**
```rust
pub async fn spawn_acp_process(
    agent_id: &str,
    cwd: &str,
    log_id: i32,
    session_id: &str,
    app_state: &std::sync::Arc<crate::db::AppState>,
) -> Result<(), String>
```

Implementation steps:
- Resolve maestro-server binary path: `which::which("maestro-server").map_err(|e| format!("maestro-server not found on PATH: {}", e))?`
- Spawn via `tokio::process::Command::new(server_path)` with `.stdin(Stdio::piped())`, `.stdout(Stdio::piped())`, `.stderr(Stdio::inherit())`, `.kill_on_drop(true)`.
- Take stdin and stdout from child BEFORE storing: `let child_stdin = child.stdin.take().expect("child stdin must be piped");` and `let child_stdout = child.stdout.take().expect("child stdout must be piped");`
- Create BufWriter wrapping child_stdin: `let mut stdin_writer = tokio::io::BufWriter::new(child_stdin);`
- Send the initial SpawnRequest to maestro-server stdin using `write_to_acp_session`:
  ```rust
  let spawn_req = MaestroRpcMessage::Request(ServerRequest::Spawn(SpawnRequest {
      agent_id: agent_id.to_string(),
      session_id: session_id.to_string(),
      cwd: cwd.to_string(),
  }));
  write_to_acp_session_raw(&mut stdin_writer, &spawn_req).await?;
  ```
- Create `let (cancel_tx, cancel_rx) = tokio::sync::oneshot::channel::<()>();`
- Create AcpProcess with child, stdin_writer, `reader_cancel_tx: Some(cancel_tx)`
- Insert into `app_state.acp_sessions.lock().await.insert(log_id, acp_process);`
- Call `spawn_reader_task(child_stdout, log_id, app_state.app_handle.clone(), std::sync::Arc::clone(app_state), cancel_rx);`
- Return `Ok(())`

**3. spawn_reader_task function (private):**
```rust
fn spawn_reader_task(
    child_stdout: tokio::process::ChildStdout,
    log_id: i32,
    app_handle: tauri::AppHandle,
    app_state: std::sync::Arc<crate::db::AppState>,
    cancel_rx: oneshot::Receiver<()>,
)
```

Implementation:
- `tokio::spawn(async move { ... })`
- Inside the task: `let mut stdout_reader = tokio::io::BufReader::new(child_stdout);`
- `let mut cancel_rx = cancel_rx;`
- Loop with `tokio::select! { biased; }`:
  - `_ = &mut cancel_rx => break` (cancel branch)
  - `result = crate::acp::transport::read_message(&mut stdout_reader) =>` match on result:
    - `Ok(MaestroRpcMessage::Response(ServerResponse::SessionUpdate(upd)))` -> `let _ = app_handle.emit(&format!("acp://session-update/{}", log_id), &upd.payload);`
    - `Ok(MaestroRpcMessage::Response(ServerResponse::TerminalOutput(out)))` -> `let _ = app_handle.emit(&format!("acp://terminal-output/{}", log_id), &out.bytes);`
    - `Ok(MaestroRpcMessage::Response(ServerResponse::PermissionRequest(req)))` -> `let _ = app_handle.emit(&format!("acp://permission-request/{}", log_id), &req);`
    - `Ok(MaestroRpcMessage::Response(ServerResponse::SpawnOk(_)))` -> `{}` (no Tauri event needed, spawn success is implied by IPC returning Ok)
    - `Ok(MaestroRpcMessage::Response(ServerResponse::Error(err)))` -> `let _ = app_handle.emit(&format!("acp://session-error/{}", log_id), &err.message);`
    - `Ok(_)` -> `{}` (ignore Request variants on stdout — wrong direction)
    - `Err(_)` -> `break` (EOF or parse error — server exited)
- After the loop: `app_state.acp_sessions.lock().await.remove(&log_id);`
- Then: `let _ = app_handle.emit(&format!("acp://session-ended/{}", log_id), ());`

**4. write_to_acp_session_raw helper (private):**
```rust
async fn write_to_acp_session_raw(
    stdin_writer: &mut BufWriter<ChildStdin>,
    msg: &MaestroRpcMessage,
) -> Result<(), String>
```
- Call `crate::acp::transport::write_message(stdin_writer, msg).await.map_err(|e| format!("write failed: {}", e))?;`

- Return `Ok(())`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Tampering | cwd argument to maestro-server | mitigate | cwd validation happens inside maestro-server/src/agent.rs (T-42-01: rejects ".." components, validates path exists). Tauri side passes cwd through without shell interpolation via Command::new + args. |
| T-43-02 | Elevation of Privilege | orphaned maestro-server processes | mitigate | kill_on_drop(true) on Child handle; removal from acp_sessions drops Child which kills subprocess. Reader task also removes from map on EOF. |
| T-43-03 | Denial of Service | oversized message from maestro-server | mitigate | read_message enforces MAX_MESSAGE_SIZE (16 MB) guard — established in Phase 41 (maestro-protocol). |
| T-43-04 | Tampering | agent_id used as Command arg | mitigate | agent_id passed as SpawnRequest.agent_id field (serialized JSON), never interpolated into shell command. maestro-server uses Command::new(agent_id) which is not shell-interpreted. |
| T-43-05 | Information Disclosure | Tauri event payloads | accept | Events emitted only to the local Tauri webview (same-process IPC, not network-exposed). No PII in session updates — agent output is developer code. |
</threat_model>
