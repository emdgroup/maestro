## What Was Built

### `src-tauri/src/ipc/acp_handlers.rs` (153 lines)

Three `#[tauri::command] #[specta::specta]` async functions:

**`start_acp_session(agent_id, cwd, session_name) -> Result<i32, String>`**
- Inserts an `execution_logs` row (status='running') in a scoped DB lock
- Derives `session_id = format!("session-{}", log_id)`
- Delegates to `crate::acp::spawn_acp_process` which spawns maestro-server and starts the background reader task
- Returns `log_id` for frontend use as session key and Tauri event channel suffix

**`send_to_acp_session(log_id, message_type, content, request_id, allowed) -> Result<(), String>`**
- Strict match on `message_type`: "prompt" or "permission_response" only (T-43-06 mitigated)
- Unknown values return `Err` immediately — no passthrough
- Constructs typed `MaestroRpcMessage` and calls `crate::acp::write_to_acp_session`

**`cancel_acp_session(log_id) -> Result<(), String>`**
- Sends `CancelRequest` best-effort (ignored if session gone)
- Removes `AcpProcess` from `acp_sessions` map (drops Child with kill_on_drop — T-43-08 mitigated)
- Sends oneshot cancel token to background reader task
- Updates `execution_logs` status to 'cancelled'
