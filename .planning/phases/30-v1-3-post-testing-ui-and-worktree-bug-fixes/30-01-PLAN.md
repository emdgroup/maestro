    - src/App.tsx (where currentProject.path originates)
  </read_first>
  <action>
    Investigate the "not a git repository" error chain:

    1. Trace the `repo_path` value from frontend to Rust:
       - `boardStore.executeTask(projectId, taskId, repoPath)` calls `api.spawnAgentExecution(projectId, taskId, repoPath)`
       - `spawn_agent_execution` passes `repo_path` to `create_worktree_for_task`
       - `create_worktree_for_task` calls `crate::git::create_worktree(&git_conn, ...)` with `GitConnection::Local { path: repo_path }`
       - `create_worktree_local` runs `git worktree add` with `current_dir(path)`

    2. The likely root cause: `repo_path` is being passed as an empty string or a relative path that doesn't resolve from the Tauri process's CWD. Check:
       - Whether `currentProject.path` in `App.tsx` is ever empty or undefined when passed to KanbanContext
       - Whether the path stored in the DB `projects.path` column is absolute or relative

    3. Fix approach: Add a validation check at the top of `spawn_agent_execution` in `execution_handlers.rs`:
       - If `repo_path` is empty, return `Err("repo_path is empty".to_string())`
       - If `repo_path` does not exist as a directory, return `Err(format!("repo_path does not exist: {}", repo_path))`
       - Log the received `repo_path` value: `println!("spawn_agent_execution: repo_path={}", repo_path);`

    4. Also add the same validation at the top of `create_worktree_for_task`:
       - Check `std::path::Path::new(repo_path).is_dir()` before proceeding
       - Return descriptive error: `Err(format!("Repository path is not a valid directory: {}. Ensure the project path is an absolute path to an existing git repository.", repo_path))`

    5. In `create_worktree_local` (git/mod.rs), add validation before running git:
       - Check that `std::path::Path::new(path).join(".git").exists()` OR `std::path::Path::new(path).join(".git").is_file()` (worktrees use .git files)
       - If neither exists, return `Err(format!("Not a git repository: {}. The path must contain a .git directory or file.", path))`
  </action>
