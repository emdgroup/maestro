  <action>
    Investigate the "not a git repository" error by tracing the full path flow. Per locked decision: NO upfront .git validation — fix the path so git runs in the right directory.

    **Root cause investigation:**

    The path flow is:
    1. App.tsx: `currentProject.path` (from DB projects.path column) -> `KanbanProvider projectPath={currentProject.path}`
    2. KanbanContext: provides `projectPath` to descendants
    3. TaskCard.tsx: `const { projectPath } = useKanban()` -> `store.executeTask(task.project_id, task.id, projectPath)`
    4. boardStore.ts: `api.spawnAgentExecution(projectId, taskId, repoPath)`
    5. execution_handlers.rs: `spawn_agent_execution(app_state, project_id, task_id, repo_path)`
    6. worktree_handlers.rs: `create_worktree_for_task(app_state, project_id, task_id, &repo_path)`
    7. git/mod.rs: `create_worktree_local(path, branch, worktree_name)` with `.current_dir(path)`

    The path stored in projects.path SHOULD be absolute (set during clone_project/open_folder). Possible failure modes:
    - The path is correct but a trailing slash or encoding issue causes git to fail
    - The path is empty string (currentProject.path could be "")
    - The path no longer exists on disk (project dir was moved/deleted)

    **Fix approach (root cause, not validation):**

    1. In `spawn_agent_execution` (execution_handlers.rs), add diagnostic logging BEFORE calling create_worktree_for_task:
       ```rust
       println!("spawn_agent_execution: repo_path='{}', is_absolute={}, exists={}",
           repo_path,
           std::path::Path::new(&repo_path).is_absolute(),
           std::path::Path::new(&repo_path).is_dir());
       ```

    2. In `spawn_agent_execution`, canonicalize the repo_path to resolve any symlinks, trailing slashes, or relative path issues:
       ```rust
       let repo_path = std::path::Path::new(&repo_path)
           .canonicalize()
           .map_err(|e| format!("Invalid repository path '{}': {}. Ensure the project directory exists.", repo_path, e))?
           .to_string_lossy()
           .to_string();
       ```
       This replaces the raw string with a verified, canonical absolute path. If the path doesn't exist, it returns a descriptive error immediately — no .git check needed.

    3. Apply the same canonicalize fix in `create_worktree_for_task` (worktree_handlers.rs) as a safety net:
       ```rust
       let repo_path_canon = std::path::Path::new(repo_path)
           .canonicalize()
           .map_err(|e| format!("Invalid repository path '{}': {}. Ensure the project directory exists.", repo_path, e))?;
       let repo_path = repo_path_canon.to_string_lossy().to_string();
       ```
       Then use `&repo_path` for all subsequent operations in the function (git_conn, abs_path computation, etc.).

    4. Also apply canonicalize in `resume_agent_execution` which has the same pattern.

    This approach fixes the root cause: if the path is relative, has symlinks, or has encoding oddities, `canonicalize()` resolves it to the real absolute path. If the path doesn't exist at all, the error message is clear and descriptive.
  </action>
