## Gray area decisions

### D-1: Row indicator — how to show classification

**Decision:** Pill/badge in each ticket row.

- `Available` → neutral/muted pill (no icon)
- `Imported` → green pill
- `Changed` → amber pill with "!" indicator

Classification text rendered as small badge next to title in the list row.
No icon-only approach — text badge is scannable without hover.

---

### D-2: Imported tab — what to show when fetch fails

**Decision:** Show imported tasks from DB even when remote fetch fails.

The "Imported" tab content comes from `tasks` (TanStack Query cache), not from the live
remote fetch. If `useFetchRemoteIssuesQuery` errors, the Imported tab still shows all
imported tasks. The "Changed" tab shows a fallback empty state with the error message
(cannot compute diffs without remote data). Available tab shows the error state.

### D-4: Fetch failure — empty state

**Decision:** Show error inline, don't block modal.

When `useFetchRemoteIssuesQuery` fails:
- Modal opens normally
- Available tab: show error card with message + Retry button
- Changed tab: show "Cannot detect changes — remote fetch failed" + Retry button
- Imported tab: works normally (DB-only)
- No full-screen error; no toast; error is scoped to the tabs that need remote data

## Additional decisions

### Priority field in RemoteIssue

**Decision:** Add `priority: Option<String>` to `RemoteIssue`.

Normalized to Maestro values (`"Urgent"` | `"High"` | `"Medium"` | `"Low"` | `null`)
before returning from each fetch function.

Mapping:
- **Linear:** 0→`null`, 1→`"Urgent"`, 2→`"High"`, 3→`"Medium"`, 4→`"Low"`
- **Jira Cloud:** `"Highest"`→`"Urgent"`, `"High"`→`"High"`, `"Medium"`→`"Medium"`, `"Low"`/`"Lowest"`→`"Low"`, else `null`
- **AzDo:** 1→`"Urgent"`, 2→`"High"`, 3→`"Medium"`, 4→`"Low"`, else `null`
- **GitHub / GitLab / Forgejo:** always `null` (no native priority)

On import: `null` priority → `TaskPriority::None`.

### AzDo HTML→markdown

**Decision:** Convert `System.Description` HTML→markdown in `azure_devops.rs`.

Use `htmd` crate (or `html2text` if htmd not suitable). Same pattern as Jira's `jc_adf`
conversion. All providers must deliver markdown to `RemoteIssue.body` — this is an internal
contract. AzDo currently violates it; fix in Phase 56.

### Assignee field

**Deferred to future phase.** User envisions: assignee = either an existing Agent profile
or a custom (agent + model + skills + MCP) configuration. Needs dedicated DB column, Rust
struct change, UI to pick/configure. Not in scope for Phase 56.

### Attachments

**Deferred to dedicated discussion.** Provider fetch scope and DB design TBD.

`external_url` opens in browser (Tauri `open::that`). Icon derived from `import_source`.
This is a Phase 56 plan detail, not a Phase 56 prerequisite.
