## Layout Specification

### "Import tickets" button — BacklogView header

Placed in the `<div className="flex items-center justify-between px-4 py-2 border-b ...">` header row, to the left of the existing "Add Task" button.

```
[  Backlog title + subtitle  ]  [ Import tickets ]  [ Add Task ]
```

- Button: `variant="outline" size="sm" className="h-8"` — secondary to the "Add Task" accent button
- Icon: `<Download className="w-4 h-4" />` (or `<ArrowDownToLine />`) from Lucide
- Visibility: rendered only when a ticketing integration is connected (gate on `ticketing_config` presence); hidden entirely (not disabled) when no integration is connected
- Source decision: REQUIREMENTS.md IMPT-01

### ImportTicketsModal — Dialog

```
DialogContent: flex flex-col gap-0 p-0 h-[80vh] sm:max-w-2xl
├── Header strip (shrink-0, border-b, px-6 py-4)
│   ├── Left: DialogTitle "Import tickets" + provider name subtitle
│   └── Right: Refresh button (icon-sm, ghost) + DialogClose (X, ghost)
├── Tab bar (shrink-0, px-6 pt-2, bg-muted/30, border-b)
│   └── LayoutGroup animated tab pills: Available | Imported | Changed
│       └── motion.span layoutId="import-modal-active-pill" (ProjectPicker pattern)
├── Tab content (flex-1, min-h-0, overflow-hidden)
│   ├── Available tab
│   │   ├── Filter bar (shrink-0, px-4 py-2, border-b): label filter Popover trigger
│   │   └── ScrollArea (flex-1): checkbox rows
│   ├── Imported tab
│   │   └── ScrollArea (flex-1): read-only rows with external URL link
│   └── Changed tab
│       └── ScrollArea (flex-1): rows with "Update task" + "Dismiss change" buttons
└── Footer (shrink-0, border-t, px-6 py-4)
    ├── Available tab: left = selection count ("N selected"), right = "Import Selected" button
    ├── Imported tab: empty (no footer actions)
    └── Changed tab: empty (no footer actions)
```

### Ticket row — Available tab

```
[ Checkbox ] [ title (flex-1, truncate) ] [ labels ] [ "Available" badge ]
```

- Row: `flex items-center gap-3 rounded-lg border px-4 py-2 transition-colors`
- Checked state: `bg-accent/10 border-accent/40` (matches BacklogView selected row)
- Unchecked: `bg-card hover:bg-accent/5`
- Labels: rendered as small `text-xs px-2 py-1 rounded bg-muted text-muted-foreground` pills, max 3 shown then "+N more"
- Classification badge: right-most element in row

### Ticket row — Imported tab

```
[ title (flex-1) ] [ labels ] [ "Imported" badge ] [ ExternalLink icon-button ]
```

- No checkbox; no actions other than external URL link
- ExternalLink: `variant="ghost" size="icon-xs"` — opens `external_url` via Tauri `open::that`
- Tooltip on ExternalLink: "Open in browser"

### Ticket row — Changed tab

```
[ title (flex-1) ] [ labels ] [ "Changed" badge ] [ "Update task" button ] [ "Dismiss change" button ]
```

- "Update task": `variant="outline" size="xs"` — performs non-destructive overwrite
- "Dismiss change": `variant="destructive" size="xs"` — clears change flag without content update; no confirmation dialog (low-stakes action, reversible by re-fetching)
- Both buttons: `disabled` while mutation is pending

Source: REQUIREMENTS.md CHNG-02, CONTEXT.md D-1

## Interaction Contracts

### Tab switching

- Uses Framer Motion `LayoutGroup` with animated pill indicator, identical to `ProjectPicker.tsx` implementation
- `layoutId="import-modal-active-pill"` on `motion.span`
- Tab slide direction: track prev/next index for `AnimatePresence` custom direction
- Tab animation: `spring, stiffness 400, damping 35` — matches ProjectPicker constants
- Available/Imported/Changed badge counts shown in parentheses next to each tab label: e.g. "Available (12)"

### Refresh button

- Icon: `<RefreshCw className="size-4" />`, applies `animate-spin` class while `isLoading` is true
- Ghost icon button: `variant="ghost" size="icon-sm"`
- Tooltip: "Refresh tickets" (`side="bottom" sideOffset={8}`)
- Triggers `useFetchRemoteIssuesQuery` refetch
- Auto-refresh: `refetchInterval: 5 * 60 * 1000` (5 minutes) while modal is open; paused when modal closes

Source: REQUIREMENTS.md IMPT-04, IMPT-05

### Label filter (Available tab only)

- Trigger: `variant="outline" size="sm"` button with `<Filter className="size-4" />` icon and "Filter" label
- PopoverContent: `w-56` list of label checkboxes; each label from the fetched remote issue set
- Multi-select: multiple labels can be active simultaneously; rows shown only if they include at least one selected label
- When no labels selected: all rows shown (filter inactive)
- Active filter state: button shows `text-accent` color and count badge overlay

Source: REQUIREMENTS.md IMPT-06

### Checkbox + bulk select

- Individual row checkboxes: Checkbox component, `data-checked` state
- "Select all" header checkbox (indeterminate state supported via `aria-checked="mixed"`) — renders above the row list, inside the Available tab scroll container header
- "Import Selected" button: disabled when no rows checked; shows pending spinner while mutation runs
- On successful import: checked rows transition from Available to Imported tab; checkbox state resets

### State machine per fetch cycle

| State | Available tab | Imported tab | Changed tab |
|-------|--------------|--------------|-------------|
| Loading | Spinner centered | Shows cached DB tasks | Spinner centered |
| Success | Checkbox list | Read-only list | Change list or empty |
| Error | Error card + "Try again" | Shows cached DB tasks | Error card + "Try again" |
| Empty (no data) | Empty state | Empty state | Empty state |

Source: CONTEXT.md D-2, D-4

## Accessibility

- `DialogTitle` present and contains "Import tickets" (required for Base UI Dialog accessibility)
- `aria-label="Close"` on `DialogClose` button
- `aria-label="Refresh tickets"` on refresh icon button (supplemental to tooltip)
- Tab panel role: use `role="tabpanel"` on each tab content area; `aria-labelledby` pointing to the tab button
- Checkbox rows: `<label>` wrapping checkbox + title text so full row click area toggles checkbox
- "Select all" checkbox: `aria-label="Select all available issues"`
- Keyboard: Tab key navigates through checkboxes; Space toggles; Enter triggers focused button
- `prefers-reduced-motion`: inherited from existing `index.css` rule — animations suppressed automatically
