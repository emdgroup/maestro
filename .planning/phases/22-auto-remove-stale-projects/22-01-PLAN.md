# Plan 22-01: Validate Project Paths in get_connection_projects (Local + SSH)

## Objective

Modify the `get_connection_projects` IPC handler to validate that each project's path still exists before returning the list. Stale projects are deleted from the database silently. Works for both local connections (using `std::fs`) and SSH connections (using the active SSH session). The frontend requires no changes.

## Context

- **File**: `src-tauri/src/ipc/project_handlers.rs`
- **Function**: `get_connection_projects(app_state, connection_id: Option<i32>)`
- **Current behaviour**: fetches all projects for a connection and returns them as-is
- **Target behaviour**: after fetching, validate each path and delete+exclude any that no longer exist

**Why SSH is feasible here:** when the user navigates to the project list for a remote connection, they have already authenticated and an SSH session is active in `AppState`. The session's `execute_command` can run `test -d "/path"` to check remote directory existence.

**Mutex safety:** `app_state.db` is a `Mutex<Connection>`. Holding this lock while doing async SSH I/O would block other IPC calls. The solution is to **drop the db lock** after fetching projects, do path validation (sync or async), then **re-acquire the lock** to delete stale entries.

**Making the function async:** `get_connection_projects` is currently a synchronous `pub fn`. To call `app_state.get_ssh_session(id).await`, it must become `async`. Tauri IPC commands support async natively — no other changes are required by this switch.

## Tasks

### Task 1 — Make `get_connection_projects` async and restructure to drop db lock before validation

Current structure (simplified):
```rust
pub fn get_connection_projects(app_state: State<Arc<AppState>>, connection_id: Option<i32>) -> Result<Vec<Project>, String> {
    let conn = app_state.db.lock()...;
    // query...
    let projects = stmt.query_map(...)...collect()?;
    Ok(projects)
}
```

New structure — drop `conn` before async work, re-acquire to delete:
```rust
pub async fn get_connection_projects(app_state: State<'_, Arc<AppState>>, connection_id: Option<i32>) -> Result<Vec<Project>, String> {
    // ── Step 1: fetch projects (db lock held briefly, then dropped) ──────────
    let projects: Vec<Project> = {
        let conn = app_state.db.lock().map_err(|e| format!("Lock failed: {}", e))?;
        let (query, params): (&str, &[&dyn ToSql]) = match connection_id {
            Some(id) => ("SELECT * FROM projects WHERE connection_id = ? ORDER BY created_at DESC", params![id]),
            None     => ("SELECT * FROM projects WHERE connection_id IS NULL ORDER BY created_at DESC", params![]),
        };
        let mut stmt = conn.prepare(query).map_err(|e| e.to_string())?;
        stmt.query_map(params, Project::from_row)
            .map_err(|e| e.to_string())?
            .collect::<Result<Vec<_>, _>>()
            .map_err(|e| e.to_string())?
        // conn drops here — lock released before async work below
    };

    // ── Step 2: validate paths ───────────────────────────────────────────────
    let stale_ids = collect_stale_project_ids(&projects, connection_id, &app_state).await;

    // ── Step 3: delete stale projects and return filtered list ───────────────
    if !stale_ids.is_empty() {
        println!(
            "get_connection_projects: removing {} stale project(s): {:?}",
            stale_ids.len(), stale_ids
        );
        let conn = app_state.db.lock().map_err(|e| format!("Lock failed: {}", e))?;
        for id in &stale_ids {
            conn.execute("DELETE FROM projects WHERE id = ?", [id])
                .map_err(|e| e.to_string())?;
        }
    }

    Ok(projects.into_iter().filter(|p| !stale_ids.contains(&p.id)).collect())
}
```

### Task 2 — Add `collect_stale_project_ids` helper function

Add a private async helper in the same file, below `get_connection_projects`:

```rust
/// Returns the IDs of projects whose paths no longer exist.
/// - Local connections: checks path existence with std::fs.
/// - SSH connections: runs `test -d` via the active SSH session.
///   If no session is found (should not happen in normal flow), skips validation.
async fn collect_stale_project_ids(
    projects: &[Project],
    connection_id: Option<i32>,
    app_state: &Arc<AppState>,
) -> Vec<i32> {
    match connection_id {
        // ── Local: synchronous filesystem check ─────────────────────────────
        None => projects
            .iter()
            .filter(|p| !std::path::Path::new(&p.path).exists())
            .map(|p| p.id)
            .collect(),

        // ── SSH: check via active session ────────────────────────────────────
        Some(conn_id) => {
            let session = match app_state.get_ssh_session(conn_id).await {
                Some(s) => s,
                None => {
                    println!("collect_stale_project_ids: no SSH session for connection {}, skipping validation", conn_id);
                    return vec![];
                }
            };

            let mut stale = Vec::new();
            for project in projects {
                // Escape any double-quotes in the path to prevent shell injection
                let safe_path = project.path.replace('"', "\\\"");
                let cmd = format!("test -d \"{}\" && echo ok || echo missing", safe_path);
                match session.execute_command(&cmd).await {
                    Ok(output) => {
                        if output.trim() != "ok" {
                            stale.push(project.id);
                        }
                    }
                    Err(e) => {
                        // On command error, err on the side of caution: keep the project
                        println!("collect_stale_project_ids: failed to check path '{}': {}", project.path, e);
                    }
                }
            }
            stale
        }
    }
}
```

**Import note:** `Arc` is already used in this file. `AppState` is already imported via `crate::db::AppState`. No new imports are needed.
