## Interaction Contracts

### Action Bar Layout

Container: `h-12 border-b border-border bg-muted/30 flex items-center px-4 gap-2 shrink-0`

All controls left-aligned in a single horizontal row, in this order:
1. Search input (flex-grow, min-w-0)
2. Priority filter button (fixed width, shrink-0)
3. Label filter button (fixed width, shrink-0)

Search input:
- Placeholder: "Search tasks..."
- Width: fills available space with `flex-1 min-w-0`
- Height: 32px (`h-8`) — compact to fit `h-12` action bar
- Icon: `Search` (lucide, 14px, muted-foreground) placed inside input via relative/absolute positioning or `InputGroup` wrapper

Priority filter trigger:
- Label when nothing selected: "Priority"
- Label when selections active: "Priority" + Badge showing count (e.g. `Priority · 2`)
- Variant: `outline`, size: `sm`
- Badge: `bg-accent text-accent-foreground` inline at end of button text

Label filter trigger:
- Label when nothing selected: "Label"
- Label when selections active: "Label" + Badge showing count
- Variant: `outline`, size: `sm`
- Same badge pattern as Priority

### Priority Popover

Trigger alignment: `align="start"` (opens below trigger, left-aligned)

Content width: 200px (`w-50` or `w-[200px]`)

Content structure:
```
┌─────────────────────┐
│ ☐ Urgent            │
│ ☐ High              │
│ ☐ Medium            │
│ ☐ Low               │
│ ☐ None              │
│─────────────────────│
│ Clear               │
└─────────────────────┘
```

Each row: `flex items-center gap-2 px-1 py-1.5 cursor-pointer hover:bg-muted rounded-sm`

Checkbox: shadcn `Checkbox`, 16px, checked when priority in `selectedPriorities`

"Clear" link:
- Renders only when `selectedPriorities.length > 0`
- `text-sm text-muted-foreground hover:text-foreground cursor-pointer`
- Separated by `border-t border-border mt-1 pt-1`

### Label Popover

Identical pattern to Priority popover.

Labels derived at render time: `[...new Set(tasks.flatMap(t => t.labels))].sort()`

Empty state (no labels on any task): show single disabled row "No labels" in `text-muted-foreground`.

Content width: 200px. Scrollable if label list exceeds 8 items: `max-h-48 overflow-y-auto`.

### Filter Logic

Filter composes with AND. Applied in `KanbanView.tsx` before passing tasks to `BoardView`:
- Query empty → all tasks pass title filter
- No selected priorities → all tasks pass priority filter
- No selected labels → all tasks pass label filter

`BoardView` receives pre-filtered `tasks` prop (filtered task list).

### 5-Column Board

`BOARD_STATUSES` array: `["Backlog", "Ready", "InProgress", "Review", "Done"]`

`COLUMN_TITLES` map adds: `Backlog: "Backlog"`

Grid: `grid-cols-5` (changed from `grid-cols-4`)

Backlog column behavior:
- Uses same `KanbanColumn` component, no special treatment
- Does NOT filter `archived_at` (only Done does that — existing behavior unchanged)

`getColumnBorderColor` and `getBadgeColor` already handle `"Backlog"` in `KanbanColumn.tsx` — no changes needed there.

### Loading State

Existing: `<div className="kanban-board">Loading tasks...</div>`

This phase does not change the loading state. It remains as-is.

- Keyboard navigation within popover: baseline only (Tab/Shift-Tab between checkboxes) — no roving tabindex required (out of scope per CONTEXT.md)
