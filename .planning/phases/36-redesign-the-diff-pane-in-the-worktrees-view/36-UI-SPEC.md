## Color

All values are CSS custom properties from `src/index.css`. Hex equivalents shown for reference — actual implementation uses the token names.

| Role | CSS Token | Usage |
|------|-----------|-------|
| Dominant (60%) | `bg-background` | Main content area background |
| Secondary (30%) | `bg-card` / `bg-muted/30` | Worktree sidebar (`bg-card`), detail header (`bg-muted/30`) |
| Accent (10%) | `border-ring` / `bg-muted/20` | Selected file entry left border + background highlight |
| Destructive | `text-destructive` | Deletion count in file list stats and per-file header (`-N`) |
| Success | `text-success` | Insertion count in file list stats and per-file header (`+N`) |
| Warning | `text-warning` | Running badge, zombie badge — unchanged from current |
| Muted foreground | `text-muted-foreground` | Modified status letter (`M`), inactive file basenames, secondary info |

Accent reserved for:
- Left border of the **selected file** in the file list (`border-ring border-l-2`)
- Selected file background tint (`bg-muted/20`)
- Matches the existing worktree sidebar selected state exactly

Status letter colors:
- `A` (Added): `text-success` — green, signals addition
- `D` (Deleted): `text-destructive` — red, signals removal
- `M` (Modified): `text-muted-foreground` — neutral, default modification

Primary visual anchor: the diff body column — the per-file header bar orients the user before the diff content fills the remainder of the pane.

## Interaction Contract

### File list selection

| State | Classes |
|-------|---------|
| Unselected, default | `border-l-2 border-transparent hover:bg-muted/10 transition-colors cursor-pointer` |
| Selected (active) | `border-l-2 border-ring bg-muted/20` |

Behavior:
- Clicking a file entry calls `setSelectedFileIndex(i)`
- Auto-select: `useEffect` sets `selectedFileIndex = 0` whenever `diffFiles` identity changes (handles worktree switch + empty reset)
- `selectedFile = selectedFileIndex !== null ? (diffFiles[selectedFileIndex] ?? null) : null`

### DiffTarget — always Head

- Module-level constant declared outside the component: `const DIFF_TARGET_HEAD: DiffTarget = { type: "Head" };`
- Passed directly to `useWorktreeDiffQuery(selectedWorktree?.id ?? null, DIFF_TARGET_HEAD)`
- Avoids query key churn on every render (no new object reference per render)

Source: RESEARCH.md Pitfall 2 + Pattern from worktree.service.ts query key structure

### States

| State | What to render |
|-------|----------------|
| No worktree selected | Unchanged: `"Select a worktree to view details"` centered in right panel |
| Worktree selected, diff loading | File list panel shows "Loading..." (text-xs text-muted-foreground py-8 text-center); diff body shows `<DiffViewer diffFile={null} loading={true} />` |
| Worktree selected, no uncommitted changes (`git_status === ""`) | File list panel shows empty (no entries); diff body shows "No uncommitted changes" centered |
| Worktree selected, files changed | File list shows entries; first file auto-selected; diff body shows selected file via DiffViewer |
| Diff query error | File list shows entries (or empty); diff body passes `error={String(diffError)}` to DiffViewer |

## Per-File Header Bar Contract

Appears above the diff body when a file is selected. Disappears when no file is selected (replaced by the "No uncommitted changes" or loading state).

Content: `{fullRelativePath}  {statusLetter}  {+insertions} {-deletions}`

Example: `src/components/execution/DiffViewer.tsx  M  +12 -4`

Layout: `flex items-center gap-2 text-xs`
- Full path: `<span className="font-mono text-foreground truncate flex-1">{selectedFile.fileName}</span>`
- Status letter: `<span className={cn("font-medium shrink-0", statusColor)}>{status}</span>`
  - A → `text-success`
  - D → `text-destructive`
  - M → `text-muted-foreground`
- Insertion count (only if > 0): `<span className="text-success shrink-0">+{insertions}</span>`
- Deletion count (only if > 0): `<span className="text-destructive shrink-0">-{deletions}</span>`

Source: CONTEXT.md per-file diff header decision + RESEARCH.md Pattern 4

## File List Entry Contract

Each entry in the 200px file list panel:

```
┌────────────────────────┐
│ M  DiffViewer.tsx      │  ← gap-2, text-xs
│    +12 -4              │  ← pl-3, mt-1, text-xs
└────────────────────────┘
```

Row 1 (`flex items-center gap-2`):
- Status letter: `text-xs font-medium shrink-0` + status color (A=success, D=destructive, M=muted-foreground)
- Basename (`fileName.split("/").pop() ?? fileName`): `text-xs font-mono truncate`

Row 2 (`text-xs mt-1 pl-3`):
- Insertions: `<span className="text-success">+{insertions}</span>` (only if > 0)
- Deletions: `<span className="text-destructive ml-1">-{deletions}</span>` (only if > 0)
- If both 0 (empty file rename): row 2 omitted

## Utility Function Contract

### `computeFileStats(hunks: string[]): { insertions: number; deletions: number }`

Location: Extract to `src/utils/helpers/diff-utils.ts` (alongside `parseDiffString`); add test cases to `src/utils/helpers/diff-utils.test.ts`.

Logic:
- Count lines starting with `+` but not `+++` → insertions
- Count lines starting with `-` but not `---` → deletions

Source: RESEARCH.md Per-file stat extraction section

### `parseDiffString` extension — add `status` field

Add `status?: "A" | "M" | "D"` to `DiffFileWithName` type in `src/types/review.ts`.

Detection logic inside `parseDiffString` in `src/utils/helpers/diff-utils.ts`:
- Line contains `new file mode` → status = `"A"`
- Line contains `deleted file mode` → status = `"D"`
- Otherwise → status = `"M"`

Default when `status` is absent (backwards compat): treat as `"M"`.

Source: RESEARCH.md Deriving M/A/D status section + Open Question 1 recommendation
