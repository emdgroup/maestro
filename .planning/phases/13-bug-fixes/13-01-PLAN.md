  <name>Task 3: Add automated bundle verification script to detect mock code in production builds</name>
  <files>
    scripts/verify-bundle.mjs
    package.json
  </files>
  <action>
Create a build-time verification script that fails if mock code is detected in production bundle (prevents regression):

1. Create scripts/verify-bundle.mjs:
   ```javascript
   // Automated bundle verification for mock code exclusion
   import { execSync } from 'child_process';
   import fs from 'fs';
   import path from 'path';

   const MOCK_MARKERS = [
     'mockDB',
     'Mock Tauri API',
     'browser-only development',
     'mock invoke',
   ];

   const BUNDLE_DIR = path.join(process.cwd(), 'dist/assets');

   console.log('Verifying production bundle for mock code...');

   if (!fs.existsSync(BUNDLE_DIR)) {
     console.error('ERROR: dist/assets not found. Run pnpm build first.');
     process.exit(1);
   }

   const jsFiles = fs.readdirSync(BUNDLE_DIR).filter(f => f.endsWith('.js'));
   let mockCodeFound = false;

   for (const marker of MOCK_MARKERS) {
     for (const file of jsFiles) {
       const filePath = path.join(BUNDLE_DIR, file);
       const content = fs.readFileSync(filePath, 'utf-8');
       if (content.includes(marker)) {
         console.error(`❌ FAILED: Found mock marker "${marker}" in ${file}`);
         mockCodeFound = true;
       }
     }
   }

   if (mockCodeFound) {
     console.error('ERROR: Mock code detected in production bundle. Did you forget import.meta.env.DEV?');
     process.exit(1);
   }

   console.log('✓ PASSED: Production bundle verified - no mock code detected');
   process.exit(0);
   ```

2. Update package.json build script:
   - Find the line: `"build": "vite build"`
   - Change to: `"build": "vite build && node scripts/verify-bundle.mjs"`
   - This ensures bundle verification runs after every production build and fails if mock code found

3. Test the script:
   - Run `pnpm build`
   - Script should complete with "✓ PASSED: Production bundle verified"
   - If any mock markers found, build fails with ERROR message
   - Verify intentional failure: temporarily add a mock marker to verify script catches it, then remove

4. Why this approach:
   - Automated verification prevents silent regressions (future devs might revert mock exclusion by accident)
   - Fails fast during build, before bundle is used
   - Script is simple and maintainable (string search, not complex AST parsing)
   - Integrates into standard build process
