## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, attachment row icon spacing |
| sm | 8px | Inline button padding, compact list items, action bar button gaps |
| md | 16px | Section padding, content area horizontal padding, sidebar field rows |
| lg | 24px | Action bar height padding, section heading bottom margin |

Exceptions:

Source: Tailwind default 4px scale; action bar height derived from existing header patterns in codebase.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 12px (text-xs) | 400 (normal) | 1.4 |
| Heading (section) | 14px (text-sm) | 600 (semibold) | 1.4 |
| Display (title field) | 20px (text-xl) | 600 (semibold) | 1.3 |

Notes:
- Body (14px/400) matches the global `body { font-size: 14px; line-height: 1.5; }` in `src/index.css` — confirmed standard.
- Label (12px/400) is used for sidebar field labels, attachment metadata, and status badge text — uses `text-xs` without a weight modifier.
- Section heading (14px/600) matches `text-sm font-semibold` used throughout `TaskDetail.tsx` for "Description", "Priority", etc.
- Display (20px/600) matches the existing `text-xl font-semibold` used for task title in `TaskDetail.tsx:217`. The new contenteditable title field uses this size.
- Action bar title (truncated read-only): 14px/400 — body weight, constrained to `max-w-xs` to prevent overflow.

Source: `src/index.css` CSS variables; existing component patterns in `TaskDetail.tsx` and `TaskCard.tsx`.

## Layout Contract

### Full-Screen Structure

```
┌─────────────────────────────────────────────────────────────────┐
│ ACTION BAR (h-12, bg-background, border-b border-border)        │
│  [title truncated]  [Improve]  [Interrupt]  [Execution]  [🗑/📦] [✕] │
├──────────────────────────────────┬──────────────────────────────┤
│ MAIN CONTENT (flex-1, p-6)       │ RIGHT SIDEBAR (w-60, p-4)    │
│                                  │                              │
│  [Title — contenteditable]       │  Status (Select)             │
│                                  │  Priority                    │
│  [Description — contenteditable] │  Agent                       │
│                                  │  Base Branch                 │
│  [Attachments]                   │  Labels                      │
│    - file list                   │  Auto-approve                │
│    - dropzone (Backlog only)     │  Worktree type               │
│                                  │                              │
└──────────────────────────────────┴──────────────────────────────┘
```

- Screen wrapper: `motion.div` with `absolute inset-0 bg-background flex flex-col`
- Enter animation: `initial={{ opacity: 0, x: 20 }}` → `animate={{ opacity: 1, x: 0 }}` at `PAGE_TRANSITION_DURATION` (0.25s), `PAGE_TRANSITION_EASING` ("easeInOut") from `src/utils/constants/animations.ts`
- Content area below action bar: `flex flex-1 overflow-hidden`
- Main content: `flex-1 overflow-y-auto p-6 space-y-6`
- Sidebar: `w-60 flex-shrink-0 border-l border-border overflow-y-auto p-4 space-y-4 bg-card`

Visual Hierarchy note: Primary visual anchor is the contenteditable title field (20px/600, top of main content area). All other content reads as subordinate to it. Section headings (14px/600) provide secondary anchoring within the main column.

Source: RESEARCH.md Pattern 7 (enter animation), D-02/D-03/D-05 layout decisions.

## Interaction States

### contenteditable Fields (Title and Description)

| State | Visual |
|-------|--------|
| Default (Backlog, not hovered) | No border; `outline-none`; text renders as plain text at display size |
| Hover (Backlog) | `border border-border rounded-sm` appears (1px, matches card border) |
| Focus (Backlog) | `border border-ring rounded-sm` — accent ring color |
| Read-only (non-Backlog) | No hover/focus border; `cursor-default`; text renders identically but is not editable |
| Empty (no content) | Placeholder text via `data-placeholder` CSS trick or `::before` pseudo-element: "Add a title..." / "Add a description..." in `text-muted-foreground` |

Save behavior: blur event triggers `useUpdateTask` mutation. No debounce on blur. No auto-save timer.

### Status Dropdown

| State | Visual |
|-------|--------|
| Backlog and Ready items | Fully interactive, normal foreground |
| InProgress, Review, Done, Cancelled items | `disabled` prop on `SelectItem`; rendered at `text-muted-foreground opacity-50` |
| Agent-required error (D-07) | Inline error text below Select: `"Assign an agent before marking as Ready."` in `text-destructive text-xs` |

### Action Bar Buttons

| Button | Visible When | Style |
|--------|-------------|-------|
| Improve | Always | `variant="ghost" size="icon"` + `Tooltip` — stub, `disabled` in this phase |
| Interrupt | `status === "InProgress"` | `variant="ghost" size="icon"` + `Tooltip` |
| Execution | `status === "InProgress" \|\| status === "Review"` | `variant="ghost" size="icon"` + `Tooltip` |
| Delete | `status !== "Done"` | `variant="ghost" size="icon"` with `text-destructive hover:text-destructive` + `Tooltip` |
| Archive | `status === "Done"` | `variant="ghost" size="icon"` + `Tooltip` |
| Close | Always | `variant="ghost" size="icon"` + `Tooltip` |

### Locked State Banner

When `status !== "Backlog"`, a read-only indicator appears directly below the action bar or integrated into it:
- A single-line banner: `"Read-only — task is {status}"` at `text-xs text-muted-foreground`
- Background: `bg-muted/30` with `py-1 px-6` padding
- No icon required; text is sufficient

### Attachments Dropzone

| State | Visual |
|-------|--------|
| Default | `border-2 border-dashed border-border rounded-lg p-4 text-center text-sm text-muted-foreground` |
| Drag-over | `border-ring bg-muted/20` |
| Hidden | `display: none` when `status !== "Backlog"` (removed from DOM, not disabled) |

### Interrupt Modal

Three labeled buttons in `DialogFooter`, left-to-right:

| Button | Variant | Action |
|--------|---------|--------|
| Resume Work | `variant="outline"` | Sends "resume" prompt to active session; closes modal |
| Rework | `variant="secondary"` | Calls `interruptTask` mutation; closes modal |
| Cancel Task | `variant="destructive"` | Calls cancel mutation (sets status=Cancelled + archived_at); closes modal |

Modal title: `"Interrupt the working agent?"` — 16px/600 via `DialogTitle`.

Destructive confirmation approach:
- **Delete task** (status ≠ Done): two-choice `AlertDialog` — "Delete this task? This cannot be undone." / "Delete Task" (destructive) / "Keep Task" (cancel). Use `AlertDialog` from `@/ui/alert-dialog` (two-choice is appropriate here).
- **Cancel Task** (interrupt flow): handled in the three-choice interrupt `Dialog` — no second confirmation needed (the modal itself is the confirmation step).
- **Archive task** (status = Done): no confirmation required — archive is reversible.

Source: CONTEXT.md Decisions D-02..D-08; RESEARCH.md §Anti-Patterns.
