  <action>
    Establish color system using CSS variables in Tailwind config:

    1. Update tailwind.config.ts with complete color palette:
       ```typescript
       export default {
         theme: {
           extend: {
             colors: {
               // Primary (interactive, buttons, links)
               primary: 'hsl(var(--primary) / <alpha-value>)',
               'primary-foreground': 'hsl(var(--primary-foreground) / <alpha-value>)',

               // Secondary (alternative accent, less prominent)
               secondary: 'hsl(var(--secondary) / <alpha-value>)',
               'secondary-foreground': 'hsl(var(--secondary-foreground) / <alpha-value>)',

               // Accent (system-driven, brand color)
               accent: 'hsl(var(--accent) / <alpha-value>)',
               'accent-foreground': 'hsl(var(--accent-foreground) / <alpha-value>)',

               // Destructive (danger, delete, error states)
               destructive: 'hsl(var(--destructive) / <alpha-value>)',
               'destructive-foreground': 'hsl(var(--destructive-foreground) / <alpha-value>)',

               // Muted (secondary text, disabled states, placeholders)
               muted: 'hsl(var(--muted) / <alpha-value>)',
               'muted-foreground': 'hsl(var(--muted-foreground) / <alpha-value>)',

               // Semantic colors (independent of theme)
               success: 'hsl(var(--success) / <alpha-value>)',
               warning: 'hsl(var(--warning) / <alpha-value>)',
               error: 'hsl(var(--error) / <alpha-value>)',

               // Background/card/border
               background: 'hsl(var(--background) / <alpha-value>)',
               foreground: 'hsl(var(--foreground) / <alpha-value>)',
               card: 'hsl(var(--card) / <alpha-value>)',
               'card-foreground': 'hsl(var(--card-foreground) / <alpha-value>)',
               popover: 'hsl(var(--popover) / <alpha-value>)',
               'popover-foreground': 'hsl(var(--popover-foreground) / <alpha-value>)',
               border: 'hsl(var(--border) / <alpha-value>)',
               input: 'hsl(var(--input) / <alpha-value>)',
               ring: 'hsl(var(--ring) / <alpha-value>)'
             }
           }
         }
       }
       ```

    2. Status color mapping (Kanban column indicators):

       - Ready: blue (use --primary or custom --ready: hsl(217, 91%, 60%))

       - Review: purple (custom --review: hsl(280, 85%, 65%))
       - Done: green (--success: hsl(142, 76%, 36%))

    3. Ensure hsl() format with alpha support for Tailwind opacity modifiers
    4. Run: `pnpm build` to verify Tailwind config loads correctly

       ```css

         font-family: 'Inter';
         src: url('/fonts/Inter-Regular.woff2') format('woff2');
         font-weight: 400;
         font-display: swap;
       }

       @font-face {
         font-family: 'Inter';
         src: url('/fonts/Inter-Medium.woff2') format('woff2');
         font-weight: 500;
         font-display: swap;
       }

       @font-face {
         font-family: 'Inter';
         src: url('/fonts/Inter-Semibold.woff2') format('woff2');
         font-weight: 600;
         font-display: swap;
       }

       /* FiraCode - terminal/code, monospace */
       @font-face {
         font-family: 'FiraCode';
         src: url('/fonts/FiraCode-Regular.woff2') format('woff2');
         font-weight: 400;
         font-display: swap;
       }

       @font-face {
         font-family: 'FiraCode';
         src: url('/fonts/FiraCode-Bold.woff2') format('woff2');
         font-weight: 700;
         font-display: swap;
       }
       ```

    2. Add font definitions to tailwind.config.ts:
       ```typescript
       fontFamily: {
         inter: ['Inter', 'system-ui', 'sans-serif'],
         firacode: ['FiraCode', 'Menlo', 'Monaco', 'monospace'],
         sans: ['Inter', 'system-ui', 'sans-serif'],
         mono: ['FiraCode', 'Menlo', 'Monaco', 'monospace'],
       }
       ```

    3. Import fonts in src/index.css:
       ```css
       @import './fonts.css';
       ```

    4. Set default font on body (in src/index.css):
       ```css
       @layer base {
         body {
           @apply font-inter text-sm text-foreground;
         }

         code, pre, .terminal-output {
           @apply font-firacode;
         }
       }
       ```

    5. Fallback strategy:
       - Inter: system-ui, Helvetica, Arial, sans-serif
       - FiraCode: Menlo, Monaco, Consolas, monospace
       - Ensures readable UI even if web fonts fail to load

    6. Font files (if not present, use Google Fonts CDN as interim solution):
       - Place in public/fonts/: Inter-Regular.woff2, Inter-Medium.woff2, Inter-Semibold.woff2
       - Place in public/fonts/: FiraCode-Regular.woff2, FiraCode-Bold.woff2
       - Or update @font-face to use CDN URLs if local fonts unavailable

    7. Run: `pnpm build` to verify font loading
