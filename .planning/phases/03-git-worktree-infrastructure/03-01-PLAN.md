<task type="auto">
  <name>Task 2: Implement git-manager.ts with core worktree operations</name>
  <files>
    sidecar/src/git-manager.ts
  </files>
  <action>
Implement git manager module with promise-based git worktree operations using simple-git library.

Functions to implement:

1. **createWorktree(repoPath: string, worktreeId: string, taskId: number): Promise<{path, branch}>**
   - Create worktree in .worktree-pool/{worktreeId}
   - Branch name: pool/agent-task-{taskId}
   - Command: git worktree add <path> -b <branch> main
   - Return { path, branch }
   - Catch errors and throw with descriptive message

2. **deleteWorktree(repoPath: string, worktreeId: string, branchName: string): Promise<void>**
   - Critical: CORRECT deletion order (research doc Pattern 3)
   - Step 1: git worktree remove .worktree-pool/{worktreeId} (with force if needed)
   - Step 2: git branch -D {branchName} (after worktree removed)
   - Step 3: git worktree prune (cleanup stale metadata)
   - Throw on failure at any step

3. **resetWorktree(repoPath: string, worktreeId: string): Promise<void>**
   - Restore worktree to clean main branch state
   - Commands: git -C <worktree-path> reset --hard main
   - Then: git -C <worktree-path> clean -fd
   - Used after task execution before returning to pool

4. **pruneWorktrees(repoPath: string): Promise<void>**
   - Clean up stale worktree metadata
   - Command: git worktree prune

5. **Helper: isWorktreeHealthy(repoPath: string, worktreePath: string): Promise<boolean>**
   - Check if worktree exists and is usable
   - Return true if healthy, false otherwise
   - No throw—used for health checks

Implementation patterns:
- Use simple-git API: git.worktree(['add', ...]), git.branch([...]), git.raw(['worktree', 'prune'])
- Each function is async/await based
- Error messages include context (which operation, which worktree)
- Logs key operations (create, delete, reset) for debugging

Reference: Phase 3 research doc Code Examples section (lines 343-510)

DO NOT:
- Use filesystem operations (rm -rf) for deletion—use git commands only
- Skip the prune step (leads to metadata accumulation)
- Catch and swallow errors (let them propagate for retry logic)
  </action>
  <verify>
    - cat sidecar/src/git-manager.ts shows all 5 functions defined
    - grep -n "git worktree remove" sidecar/src/git-manager.ts shows correct order (worktree before branch)
    - grep -n "async function" sidecar/src/git-manager.ts shows all functions are async/promise-based
    - grep -n "export" sidecar/src/git-manager.ts shows all functions exported as named exports
  </verify>
  <done>
