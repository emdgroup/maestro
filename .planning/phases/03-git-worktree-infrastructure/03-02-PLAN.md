Implement Tauri IPC commands for worktree lease/return with database transactions to prevent race conditions.

Add to `src-tauri/src/ipc/handlers.rs`:

1. **#[tauri::command] pub fn lease_worktree(...) -> Result<Worktree, String>**

   Parameters:
   - project_id: i32
   - task_id: i32
   - state: State<Arc<AppState>>

   Logic (with transaction):
   a. Lock database mutex
   b. Start transaction
   c. Query: SELECT * FROM worktrees WHERE project_id=? AND status='Available' LIMIT 1
   d. If found:
      - Update status to 'Leased', set leased_at = NOW()
      - Commit transaction
      - Return Worktree
   e. If not found:
      - Count total worktrees: SELECT COUNT(*) FROM worktrees WHERE project_id=?
      - If count < 5 (max pool size):
        * Generate worktree_id: wt-001, wt-002, ... (zero-padded)
        * Generate branch: pool/agent-task-{task_id}
        * Generate path: .worktree-pool/{worktree_id}
        * Insert new row: INSERT INTO worktrees (project_id, branch_name, path, status, created_at) VALUES (...)
        * Update to 'Leased' immediately
        * Commit transaction
        * Return Worktree (newly created)
      - Else:
        * Return Err("Pool exhausted, max 5 worktrees reached")

   Key points:
   - Use transaction() for atomicity (prevents two threads creating at same path)
   - chrono::Utc::now().to_rfc3339() for timestamps
   - Error messages describe why (exhausted, lock failed, query failed)

2. **#[tauri::command] pub fn return_worktree(...) -> Result<(), String>**

   Parameters:
   - worktree_id: i32
   - state: State<Arc<AppState>>

   Logic:
   - Lock database mutex
   - Update: UPDATE worktrees SET status='Available', returned_at=NOW() WHERE id=?
   - Return Ok(()) or Err(...)

Implementation reference: Phase 3 research doc lines 384-450 (Rust transaction pattern)

DO NOT:
- Skip transaction—race condition will corrupt pool state
- Hardcode pool size (use const POOL_MAX_SIZE = 5 at module level)
- Return user-facing errors without context
- Forget to unlock mutex (use map_err patterns)

**#[tauri::command] pub fn get_pool_status(...) -> Result<PoolStatus, String>**

Parameters:
- project_id: i32
- state: State<Arc<AppState>>

Logic:
- Lock database mutex
- Execute COUNT queries for each status:
  * SELECT COUNT(*) FROM worktrees WHERE project_id=? AND status='Available'
  * SELECT COUNT(*) FROM worktrees WHERE project_id=? AND status='Leased'
  * SELECT COUNT(*) FROM worktrees WHERE project_id=? AND status='InUse'
  * SELECT COUNT(*) FROM worktrees WHERE project_id=? AND status='Dirty'
- Calculate: total = sum of counts, utilization_percent = (leased + in_use) / total * 100.0
- Return PoolStatus { total, available, leased, in_use, dirty, utilization_percent }
