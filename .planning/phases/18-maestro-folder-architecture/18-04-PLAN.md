  <action>
    Create new file `src-tauri/src/db/migration.rs` with:

    1. Import statements:
       ```rust
       use rusqlite::Connection;
       use crate::models::{ProjectConfig, ProjectState, TaskSnapshot, WorktreeSnapshot};
       use crate::db::project_storage;
       use chrono::Utc;
       ```

    2. Function: `needs_migration(conn: &Connection, project_id: i32) -> Result<bool, String>`
       - Purpose: Check if project has migrated_at = NULL
       - Query: `SELECT migrated_at FROM projects WHERE id = ?`
       - Return true if migrated_at IS NULL (not migrated), false if timestamp exists
       - Return Err if project not found

    3. Function: `migrate_legacy_project(conn: &Connection, project_id: i32, project_path: &str) -> Result<(), String>`
       - Purpose: Main migration entry point (MUST be idempotent)
       - Logic:
         a. Check needs_migration() - if false, return Ok(()) immediately (already migrated)
         b. Create .maestro folder: project_storage::create_project_maestro_folder(project_path)?
         c. Load project config from database:
            - Query settings table for model_default, mcp_allowlist, skills_default
            - Create ProjectConfig with these values + current timestamp
         d. Export config: project_storage::export_config_to_settings(&config, project_path)?
         e. Load all tasks for project: Query tasks table WHERE project_id = ?
            - Convert Task rows to TaskSnapshot structs
         f. Load all worktrees for project: Query worktrees table WHERE project_id = ?
            - Convert Worktree rows to WorktreeSnapshot structs
         g. Create ProjectState with tasks and worktrees
         h. Export state: project_storage::export_state_to_file(&state, project_path)?
         i. Mark project as migrated: UPDATE projects SET migrated_at = ? WHERE id = ?
            - Use Utc::now().to_rfc3339() for timestamp
         j. Return Ok(())
       - All errors should be Err(msg) with descriptive context

    4. Function: `get_unmigrated_projects(conn: &Connection) -> Result<Vec<(i32, String)>, String>`
       - Purpose: Find all projects that haven't been migrated
       - Query: `SELECT id, path FROM projects WHERE migrated_at IS NULL`
       - Return Vec of (project_id, project_path) tuples
       - Used for bulk migration operations

    Reference: 18-RESEARCH.md Pattern 3 (Backward Compatibility Detection) and Pitfall 2 (Migration Idempotency)
  </action>

    4. migrate_legacy_project() calls project_storage functions for file operations
