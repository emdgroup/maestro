  <name>Task 1: Enhance attach_terminal handler with input command support and signal handling</name>
  <files>src-tauri/src/ipc/handlers.rs</files>
  <action>
Enhance existing attach_terminal handler to support interactive input:

1. Create new `send_terminal_input(task_id, input: String) -> Result<(), String>` IPC command
   - Look up PTY session in AppState.pty_sessions by task_id
   - Get PTY master from session
   - Write input bytes to PTY master (enables user typing and commands)
   - Handle special cases:
     - Input == "\x03" → send SIGINT signal (Ctrl+C behavior)
     - Input == "\x1a" → send SIGTERM signal (Ctrl+Z behavior)
     - Input == "\r" or "\n" → append newline and write (line buffering)
     - Regular text → write directly to PTY
   - Return error if task_id not found or PTY session missing
   - Log all user input for debugging (without passwords)

2. Enhance attach_terminal to accept optional `include_history: bool` parameter
   - If true, prepend terminal_output from execution log to channel stream
   - Send history as initial burst before live stream starts
   - Allows frontend to show full context when attaching

3. Handle PTY cleanup on detach:
   - When frontend closes terminal, it should send explicit "detach" signal (separate IPC call)
   - Don't kill PTY on detach - let it continue running
   - Stop streaming to frontend but leave PTY session alive
   - Remove channel from streams

4. Add error handling for PTY write failures:
   - If write fails, assume PTY is dead
   - Send error message to channel: "Terminal disconnected"
   - Clean up session from AppState.pty_sessions
   - Log error for debugging

Use portable-pty's write methods (already imported). Follow existing patterns in spawn_agent_cli_pty.

  </verify>
