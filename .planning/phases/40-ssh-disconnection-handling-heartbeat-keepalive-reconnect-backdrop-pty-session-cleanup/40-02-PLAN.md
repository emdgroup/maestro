<objective>
Add PTY session cleanup to the heartbeat task so that when an SSH connection is lost, all running PTY sessions for that connection are marked as failed in the database with an appropriate error_event, and their handles are removed from the ssh_pty_sessions map.

Purpose: Without cleanup, the Agents view shows PTY sessions as "running" even after the SSH connection has died. Users see stale green indicators for sessions that will never produce output again. This plan ensures the UI accurately reflects reality by marking dead sessions as failed immediately on connection loss.

Output: Modified heartbeat task in session.rs that cleans up PTY sessions on connection loss.
</objective>

**Step 2: Add a `cleanup_pty_sessions_for_connection` helper function in session.rs:**

```rust
/// Clean up all SSH PTY sessions associated with a given connection_id.
///
/// Iterates ssh_pty_sessions, finds sessions whose execution log belongs to a project
/// with the given connection_id, persists their terminal history to DB, marks them
/// as failed, and removes them from the map.
async fn cleanup_pty_sessions_for_connection(
    app_state: &Arc<crate::db::AppState>,
    connection_id: i32,
) {
    // Step 1: Collect log_ids of SSH PTY sessions to clean up.
    // SSH PTY sessions are keyed by log_id in the map. We need to identify which
    // ones belong to this connection. Since execution_logs don't store connection_id
    // directly, we identify them by checking which project's connection_id matches.
    //
    // However, interactive sessions (task_id IS NULL) have no project link via task_id.
    // For simplicity and correctness: clean up ALL SSH PTY sessions when a connection
    // is lost, because a single user typically has only one SSH connection active.
    // The ssh_pty_sessions map only contains remote PTY sessions (local PTY sessions
    // use pty_sessions, not ssh_pty_sessions).
    
    let mut log_ids_to_cleanup: Vec<i32> = Vec::new();
    let mut history_snapshots: Vec<(i32, String)> = Vec::new();

    {
        let sessions = app_state.ssh_pty_sessions.lock().await;
        // Collect all SSH PTY session log_ids and their history for persistence
        for (log_id, handle) in sessions.iter() {
            log_ids_to_cleanup.push(*log_id);
            // Persist terminal history before marking failed
            if let Ok(hist) = handle.history.try_lock() {
                if !hist.is_empty() {
                    history_snapshots.push((*log_id, hist.clone()));
                }
            }
            // Signal the reader task to stop
            handle.process_ended.store(true, std::sync::atomic::Ordering::Release);
            handle.notify.notify_one();
        }
    }

    if log_ids_to_cleanup.is_empty() {
        return;
    }

    // Step 2: Write history and mark as failed in DB with error_event
    if let Ok(conn) = app_state.db.lock() {
        let now = chrono::Utc::now().to_rfc3339();

        // Persist terminal history
        for (log_id, snapshot) in &history_snapshots {
            let _ = conn.execute(
                "UPDATE execution_logs SET terminal_output = ?1 WHERE id = ?2",
                rusqlite::params![snapshot, log_id],
            );
        }

        // Mark as failed with error_event containing ssh_connection_lost details
        for log_id in &log_ids_to_cleanup {
            let _ = conn.execute(
                "UPDATE execution_logs SET status = 'failed', completed_at = ?1, error_event = json_object('error_type', 'ssh_connection_lost', 'message', 'SSH connection lost', 'suggestions', json_array(), 'detected_at', ?1) WHERE id = ?2 AND status = 'running'",
                rusqlite::params![&now, log_id],
            );
        }
    }

    // Step 3: Remove from the map
    {
        let mut sessions = app_state.ssh_pty_sessions.lock().await;
        for log_id in &log_ids_to_cleanup {
            sessions.remove(log_id);
        }
    }
}
```
