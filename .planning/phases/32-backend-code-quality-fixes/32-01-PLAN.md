**H1 — Fix three broken queries in review_handlers.rs:**

The V5 schema has NO `status` column on `worktrees`. Worktrees are identified by `task_id` join, not status.

1. `get_diff_for_review` (around line 62): Replace the query:
```sql
SELECT path, branch_name FROM worktrees WHERE project_id = ? AND (status = 'InUse' OR status = 'Leased')
```
with:
```sql
SELECT w.path, w.branch_name FROM worktrees w WHERE w.task_id = ?
```
And pass `task_id` (not `proj_id`) as the parameter. The task_id is already available in scope.

2. `approve_task_and_merge` (around line 274): Replace the subquery:
```sql
SELECT id FROM worktrees WHERE project_id = t.project_id AND (status = 'InUse' OR status = 'Leased') LIMIT 1
```
with:
```sql
SELECT id FROM worktrees WHERE task_id = t.id LIMIT 1
```
This joins worktree to the task directly via task_id FK.

3. `finalize_successful_merge` (around line 389): Remove the line:
```rust
conn.execute(
    "UPDATE worktrees SET status = 'Dirty' WHERE id = ?",
    rusqlite::params![worktree_id],
)
```
and the associated println. The V5 schema has no `status` column on worktrees. The worktree cleanup via sidecar + DB DELETE that follows is sufficient.

**H2 — Fix .expect() panics in project_handlers.rs:**

In `clone_project` (around line 319) and `create_new_project` (around line 457), replace:
```rust
).expect("Failed to insert project");
```
with:
```rust
).map_err(|e| format!("Failed to insert project: {}", e))?;
```

Also in `create_project` (around line 457), same pattern:
```rust
).expect(&format!("Failed to insert project {}", name));
```
Replace with:
```rust
).map_err(|e| format!("Failed to insert project '{}': {}", name, e))?;
```

Since these are inside `unwrap_or_else` closures, the closure needs to return `Result`. Restructure: replace the `existing.unwrap_or_else(|| { ... })` pattern with a proper `match` on `existing` that returns `Result<i32, String>`, then use `?` on the outer expression. Specifically, change:
```rust
existing.unwrap_or_else(|| {
    // ...
    conn.execute(...).expect("...");
    conn.last_insert_rowid() as i32
})
```
to:
```rust
match existing {
    Some(id) => id,
    None => {
        // ...
        conn.execute(...).map_err(|e| format!("Failed to insert project: {}", e))?;
        conn.last_insert_rowid() as i32
    }
}
```

Apply this to all three locations: `clone_project`, `create_new_project`, and `create_project`.
