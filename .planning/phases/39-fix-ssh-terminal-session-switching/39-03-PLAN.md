<objective>
Move the tryAttach() call inside the requestAnimationFrame callback (after fitAddon.fit()) and write a clear-screen escape sequence before attaching. This ensures the terminal is blank on the first visible frame, and the SIGWINCH triggered by fit() causes the running program to repaint.

Purpose: Eliminate the brief stale-content flash when switching terminal sessions. The terminal shows blank for ~1 frame, then receives fresh repainted content from SIGWINCH.

Output: Modified Terminal.tsx with correct mount timing.
</objective>

const rafId = requestAnimationFrame(() => {

After these changes, the execution order on mount is:
1. Terminal opens, onResize handler registered
2. rAF fires before next paint
3. fitAddon.fit() runs, triggers onResize -> resizeTerminal IPC -> SIGWINCH to remote/local PTY
4. terminal.write('\x1b[2J\x1b[H') clears the xterm buffer (cosmetic guard)
5. tryAttach() fires, which calls api.attachTerminal() — new output flows from SIGWINCH repaint

The channel and its onmessage handler are set up BEFORE the rAF (lines 63-68), so data arriving from attachTerminal will be written to the terminal correctly.
