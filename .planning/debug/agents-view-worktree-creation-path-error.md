## Resolution
<!-- OVERWRITE as understanding evolves -->

root_cause: spawn_interactive_execution in execution_handlers.rs does not handle remote SSH projects. It calls canonicalize_repo_path (local Windows path canonicalization), tokio::fs::create_dir_all (local filesystem), and constructs GitConnection::Local — all of which fail when the project is on a remote Linux server. The working path (WorktreesView create_worktree) correctly looks up the project, calls get_git_connection to get SSH or local connection, and skips local filesystem ops for remote projects.
fix: Rewrite spawn_interactive_execution to: (1) skip canonicalize for remote projects, (2) look up project from DB and call get_git_connection, (3) gate create_dir_all behind is_remote check, (4) pass the proper git_conn to crate::git::create_worktree. Mirror the pattern in worktree_handlers.rs create_worktree.
verification: cargo check passes clean. Fix mirrors the SSH-aware pattern from worktree_handlers.rs create_worktree. Awaiting runtime verification on remote SSH project.
